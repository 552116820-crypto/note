• 如果你说的是“知识图谱里的实体名称去重/对齐”，只看 name embedding 的话，我建议不要只设一个阈值直接合并，而是分三档。

  假设你用的是 L2 normalize 后的 cosine similarity，可以先这样起步：

  - >= 0.92：可作为“高置信候选”，再叠加类型一致、别名规则后再自动合并
  - 0.85 ~ 0.92：人工复核或二次排序区
  - < 0.85：通常不合并

  如果实体名称很短、很泛，阈值要再提高：

  - 人名：自动合并常从 0.94 ~ 0.97 起
  - 公司名：常从 0.90 ~ 0.94 起
  - 缩写、别名很多的场景：候选召回可放宽到 0.80 ~ 0.88，但不能直接 merge

  更实用的做法是分两层：

  1. 召回阈值：0.78 ~ 0.85
  2. 自动合并阈值：0.90 ~ 0.95

  原因很简单：名称 embedding 很容易把“语义接近但不是同一实体”的名字拉近。
  比如公司、药品、职位、地名，单靠 embedding 很容易误合并。

  所以生产上更稳的是混合打分：

  - embedding similarity
  - 编辑距离 / Jaro-Winkler
  - 类型是否一致
  - 别名词典
  - 编号/域名/证件号/统一社会信用代码
  - 上下文邻居是否相似

  final_score =
  - 候选召回：0.82
  - 人工复核：0.87
  - 自动合并：0.93

  如果你有标注样本，正确方法不是问“行业通用阈值”，而是：

  - 抽 500 ~ 2000 对正负样本
  - 画相似度分布
  - 以 Precision 优先选自动合并阈值
  - 以 Recall 优先选召回阈值

  一句话建议：
  名称 embedding 阈值默认先从 0.93 自动合并、0.82 候选召回 开始试。
