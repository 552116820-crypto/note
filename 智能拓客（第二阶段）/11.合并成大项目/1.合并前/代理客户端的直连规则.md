## 可选加固：用 Parsers 显式钉死阿里云段

如果想不依赖 GEOIP 数据库、显式写死直连（也防机场以后改规则模板），**别直接编辑 profile 文件**——你的订阅开了 24 小时自动更新，改了会被覆盖。正确位置是 CFW 的 **Settings → Profiles 卡片 → Parsers → Edit**，填：

```yaml
parsers:
  - url: https://sub05.711997.xyz/api/v1/client/subscribe?token=fbcfa5d5ab08991ca39c22c0126a3c3a
    yaml:
      prepend-rules:
        - IP-CIDR,8.138.104.14/32,DIRECT,no-resolve   # 只放行这台 MySQL
        - IP-CIDR,8.128.0.0/10,DIRECT,no-resolve      # 或整个阿里云 8.x 段（8.128–8.191）
```

保存后右键「银河云-企业Mini」→ Update 重新拉取，Parsers 会在每次订阅更新时自动重新注入，所以是持久的。（顺带：CFW 的 yaml Mixin 不适合干这个——它是整键替换，会把订阅里 500 多条规则全干掉。）

## 如果你坚持留在全局模式

那 Clash 层面无解（全局无规则可言），只能在 Windows 路由表层面绕过 TUN：

```
route -p add 8.138.104.14 mask 255.255.255.255 <你的物理网关IP> metric 1
```

/32 路由比 Clash TUN 的 /1 路由更精确，会优先命中，流量直接走物理网卡。但这个方案脆（换网络环境网关变了就失效），不如切规则模式干净。
