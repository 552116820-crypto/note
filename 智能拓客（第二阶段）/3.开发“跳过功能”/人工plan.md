前端推荐的客户界面会有个“跳过”按钮，用户下次点进来，应该优先展示没有看过的客户

我的初步设想就是把"把用户状态叠加到推荐结果上"，我在amer_sales_assist数据库已经为“跳过功能”新建了表。


**软分层**：未看过的排前、跳过的沉到末尾，两层各自按分数排。好处是跳过的不会永久消失（未看过的翻完后还能兜底出现），下次进来天然优先未看过。

**在缓存"未命中、重算"那一刻**，顺手读一次该业务员的 skip 集合，把候选列表切成 `[未看过, 跳过]`（各自按分数排），把这个顺序存进缓存 blob。翻页就切这个冻结的顺序。

**写一个小接口**

```
POST /api/v1/leads/skip   { "employee_code": "000068", "customer_id": "198" }
```

顺手在 `pagination` 里加 `unseen_total` / `seen_total`，前端能显示"还有 N 个新客户"。

**跳过语义**：沉底
**"看过"靠什么标记**：只认"跳过"按钮（推荐，MVP）
**跳过要不要过期/复活**：跳过之后**永久保持沉底状态**，不会自己复活
