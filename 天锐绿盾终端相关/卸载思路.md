  复盘说明
  这次删除天锐绿盾终端，实际走的是四步：

  1. 识别真实安装位置和关键组件
     它不在普通 Program Files，而是在 C:\Inetpub\ftproot\Tipray\LdTerm。
     核心组件包括：

  - 服务：LeaderTerm、LeaderTermDaemon
  - 驱动：LdMFilter、Ldcore、ldwfp、LdCdRomFilters
  - 进程：LdTerm.exe、LdApproval.exe、LdFileGate.exe、LdMsgInject*.exe、LdTermPlug*.exe

  2. 先停用，再阻止它重启
     先把服务和驱动改成 disabled，再结束用户态进程。
     这样做的原因是：这类终端经常会自恢复，不先禁用，删文件意义不大。
  3. 优先尝试它自己的卸载器
     从安装目录里的 Setup.exe 挖出卸载参数，尝试官方卸载。
     这一步的目标不是“能不能一次成功”，而是让它先帮忙移除一部分服务、驱动和注册项。
  4. 对剩余残留做强制清理
     重启后，继续删除：

  - 没清掉的驱动/服务注册
  - HKLM\SYSTEM\CurrentControlSet\Services\ldwfp
  - C:\Inetpub\ftproot\Tipray
  - 相关 .sys 驱动文件

  为什么要重启
  因为像 LdMFilter 这种文件系统过滤驱动，常常不能在线卸载。
  它即使已经被禁用，也往往要等重启后才真正脱离系统。

  为什么最后图片不再被加密
  因为真正负责文件拦截和透明加密的，不只是前台程序，而是：

  - 文件系统过滤驱动
  - 用户态注入模块
  - 后台守护服务
    这些组件都被移除后，文件写入链路就不再被绿盾接管，所以新文件不会再被加密。

  这类软件以后怎么判断有没有卸干净
  看四件事就够了：

  - 没有相关进程
  - 没有相关服务/驱动
  - 没有安装目录
  - 没有残留 .sys 驱动文件

  这次最终确认已清干净的依据

  - 相关进程不存在
  - LeaderTerm、LeaderTermDaemon、Ldcore、ldwfp、LdCdRomFilters、LdMFilter 都显示“未安装”

  - 残留驱动文件已删除
