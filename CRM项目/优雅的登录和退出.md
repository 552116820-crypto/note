• 所有命令都在项目根目录执行：
  /home/huangch001002/projects/crm_manage_system

  从零启动

  1. 启后端容器

	  docker compose --env-file .compose.env up -d --build

  2. 清一次后端缓存

  docker compose --env-file .compose.env exec -T php php think clear
  
`exec` 表示：在已经运行的容器里面执行命令
`-T` 的意思是：禁用 TTY（伪终端）
  

  3. 第一次启动前端时，先装依赖

  docker compose --env-file .compose.env run --rm node14 npm install

  4. 启前端开发服务

  docker compose --env-file .compose.env run --rm -p 9528:9528 node14 npm run dev

  5. 打开页面

  - 后端: http://localhost:8080
  - 前端: http://localhost:9528/adminVueHtml/

  以后日常启动
  后端一般直接：

  docker compose --env-file .compose.env up -d
  docker compose --env-file .compose.env exec -T php php think clear

  前端直接：

  docker compose --env-file .compose.env run --rm -p 9528:9528 node14 npm run dev

  安全优雅退出
  前端如果是当前终端跑着的，最优雅就是直接按：

  Ctrl + C

  因为你用了 run --rm，停掉后这个临时 node14 容器会自动删掉。

  如果你已经关了终端，先找前端容器：

  docker ps --format 'table {{.Names}}\t{{.Ports}}'

  找到占用 9528 的那个，一般名字像 crm_manage_system-node14-run-xxxx，再停掉：

  docker stop 容器名

  后端如果只是暂时不用，推荐：

  docker compose --env-file .compose.env stop

  这会优雅停止容器，数据还在。

  如果想完整关闭并移除容器、网络，但保留数据库数据卷，推荐：

  docker compose --env-file .compose.env down

  不要随便用：

  docker compose --env-file .compose.env down -v

  这个会把 mysql 和 redis 的卷也删掉，等于清库。

  常用补充
  看状态：

  docker compose --env-file .compose.env ps

  看日志：

  docker compose --env-file .compose.env logs -f nginx php

  重启全部后端：

  docker compose --env-file .compose.env restart

  如果 9528 报端口占用，先停旧前端容器，再重新跑前端。



• 这是用来查看“当前正在运行的 Docker 容器列表”的命令，而且只显示你最关心的两列：

  docker ps --format 'table {{.Names}}\t{{.Ports}}'

  含义：

  - docker ps：查看正在运行的容器
  - --format：按你指定的格式输出
  - {{.Names}}：容器名
  - {{.Ports}}：端口映射
  - table：以表格形式显示
  - \t：制表符，对齐列

  你执行后通常会看到类似：

  NAMES                                   PORTS
  crm_manage_system-nginx-1               0.0.0.0:8080->80/tcp
  crm_manage_system-node14-run-xxxx       0.0.0.0:9528->9528/tcp
  crm_manage_system-mysql-1               0.0.0.0:3307->3306/tcp

  你要找的就是占用 9528 的那个前端容器，比如：

  crm_manage_system-node14-run-xxxx

  然后停止它：

  docker stop crm_manage_system-node14-run-xxxx

  如果你嫌这个命令难记，直接用这个也行：

  docker ps

  只是输出会更长、更乱一点。
