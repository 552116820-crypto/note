• 最省事的方案是：不要在本机装这些老版本，直接用 Docker Desktop + WSL2 跑一套隔离环境。本机已经装好的 Python / Node 保留着，不动；这个 CRM 项目里只用容器里的版本。

  我看了一下，你现在这套 WSL 里还没有 docker 命令，而且项目目录在 /mnt/e/mysql_connect。按 Docker 的 WSL 文档，这种场景更适合先在 Windows 装 Docker Desktop，然后把项目放到 WSL 的 Linux 文件系统里，比如 ~/
  projects/crm，不要继续放 /mnt/e/...，不然容器下文件 IO 往往更慢、权限也更容易出问题。

  怎么做

  1. 在 Windows 安装 Docker Desktop。
  2. 打开 Docker Desktop：
     Settings -> General -> Use WSL 2 based engine
  3. 再到：
     Settings -> Resources -> WSL Integration
     把你现在用的 Ubuntu/WSL 发行版打开。
  4. 把 CRM 代码移动到类似 ~/projects/crm。
  5. 在项目里写一个 compose.yaml，把长期服务都放进去：mysql、php、nginx、redis、beanstalkd，再加两个“工具容器” node14 和 py27。

  可以直接按这个思路起：

  services:
    mysql:
      image: sameersbn/mysql:5.7.26-0
      ports:
        - "3307:3306"
      environment:
        MYSQL_ROOT_PASSWORD: root123
        MYSQL_DATABASE: crm
        MYSQL_USER: crm
        MYSQL_PASSWORD: crm123
      volumes:
        - mysql57:/var/lib/mysql

    redis:
      image: redis:7-alpine
      ports:
        - "6379:6379"

    beanstalkd:
      build:
        context: ./docker/beanstalkd
      ports:
        - "11300:11300"

    php:
      image: hermsi/alpine-fpm-php:7.3.4
      working_dir: /var/www/html
      volumes:
        - ./crm:/var/www/html
      depends_on:
        - mysql
        - redis
        - beanstalkd

    nginx:
      image: nginx:alpine
      ports:
        - "8080:80"
      volumes:
        - ./crm:/var/www/html
        - ./docker/nginx/default.conf:/etc/nginx/conf.d/default.conf:ro
      depends_on:
        - php

    node14:
      image: node:14.21.3-buster
      working_dir: /work
      volumes:
        - ./crm:/work
      command: ["sleep", "infinity"]
      profiles: ["tool"]

    py27:
      image: python:2
      working_dir: /work
      volumes:
        - ./crm:/work
      command: ["sleep", "infinity"]
      profiles: ["tool"]

  volumes:
    mysql57:

  beanstalkd 没必要特意找老镜像，自己建一个最省心：

  FROM debian:bullseye-slim
  RUN apt-get update && apt-get install -y --no-install-recommends beanstalkd \
   && rm -rf /var/lib/apt/lists/*
  CMD ["beanstalkd", "-l", "0.0.0.0", "-p", "11300"]

  启动后常用命令就是：

  docker compose up -d
  docker compose exec php php -v
  docker compose run --rm node14 node -v
  docker compose run --rm py27 python -V
  docker compose exec mysql mysql -uroot -p

  几个关键点

  - php 7.3.4 nts 在 Linux 容器里本质上就是选非 zts 版本，别用带 zts 的镜像。
  - MySQL 5.7.26 如果 CRM 真卡死这个小版本，开发环境用上面这种精确 tag 最直接；如果只要求 5.7 大版本，也可以换 Oracle 官方的 mysql/mysql-server:5.7。
  - 你本机已有的 Python / Node 不用卸，也不用切版本；这个 CRM 项目里只用 docker compose run/exec。
  - 这些版本都比较老，建议只用于本地开发，不要直接暴露到公网。
