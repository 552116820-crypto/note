• 可以，按这套仓库的配置，改 .env 就够了，不用改代码，也不用重建镜像。

  改哪里

  - 主业务库走 [DATABASE]，模板在 .env.docker.example:7
  - NCC SQL Server 走 [NCC_DATABASE]，模板在 .env.docker.example:51
  - 连接映射在 config/database.php:23
  - php 容器已经支持 host.docker.internal，见 compose.yaml:58

  最常见做法：把整套本地应用切到真实 CRM MySQL
  把本地 .env 的 [DATABASE] 改成真实库：

  [DATABASE]
  DRIVER=mysql
  TYPE=mysql
  HOSTNAME=你的真实数据库地址
  DATABASE=你的数据库名
  USERNAME=你的账号
  PASSWORD="你的密码"
  HOSTPORT=3306
  CHARSET=utf8mb4
  PREFIX=

  如果部分历史命令还会走“正式库连接名”，把 [DATABASE_ZHENGSHI] 也同步改掉：

  [DATABASE_ZHENGSHI]
  TYPE=mysql
  HOSTNAME=你的真实数据库地址
  DATABASE=你的数据库名
  USERNAME=你的账号
  PASSWORD="你的密码"
  HOSTPORT=3306
  CHARSET=utf8mb4
  PREFIX=

  如果还要连真实 NCC
  改 .env 里的 [NCC_DATABASE]：

  [NCC_DATABASE]
  HOSTNAME=你的SQLServer地址
  DATABASE=你的库名
  USERNAME=你的账号
  PASSWORD="你的密码"
  HOSTPORT=1433

  如果真实库只能通过本机 VPN / SSH 隧道访问
  把隧道开在宿主机上，然后容器里把主机写成 host.docker.internal。比如你本机开了 127.0.0.1:3308 -> 真实MySQL，那 .env 里写：

  [DATABASE]
  HOSTNAME=host.docker.internal
  HOSTPORT=3308
  DATABASE=你的数据库名
  USERNAME=你的账号
  PASSWORD="你的密码"
  CHARSET=utf8mb4

  这个仓库已经给 php 容器配了 host.docker.internal，不需要再改 Compose。

  改完后怎么生效

  docker compose --env-file .compose.env restart php nginx
  docker compose --env-file .compose.env exec php php think Clear

  怎么验证连通
  先验证 PHP 容器能连上当前 .env 里的 MySQL：

  docker compose --env-file .compose.env exec php php -r '
  $c=parse_ini_file("/var/www/html/.env", true)["DATABASE"];
  $dsn="mysql:host={$c["HOSTNAME"]};port={$c["HOSTPORT"]};dbname={$c["DATABASE"]};charset=".($c["CHARSET"] ?: "utf8mb4");
  new PDO($dsn, $c["USERNAME"], $c["PASSWORD"]);
  echo "mysql ok\n";
  '

  如果要验 NCC：

  docker compose --env-file .compose.env exec php php -r '
  $c=parse_ini_file("/var/www/html/.env", true)["NCC_DATABASE"];
  $dsn="sqlsrv:Server={$c["HOSTNAME"]},{$c["HOSTPORT"]};Database={$c["DATABASE"]}";
  new PDO($dsn, $c["USERNAME"], $c["PASSWORD"]);
  echo "sqlsrv ok\n";
  '

  这次报表要注意的一点

  - 报表接口现在走的是默认 mysql 连接，也就是 config/database.php:23 这一段。
  - 只改 [DATABASE_ZHENGSHI]，这张报表不会切到真实数据。
  - 想让“整套后台”都看真实数据，就改 [DATABASE]。
  - 想让“只有这张报表”看真实库、其他功能继续走本地库，需要我把这张报表改成显式使用 mysqlzhengshi 连接。

  两个实际限制

  - 真实库必须放行你当前机器的 IP，或者你先连好 VPN/堡垒机/隧道。
  - 即使应用改连真实库，mysql 本地容器最好先别停，因为 php 服务当前仍依赖它启动，见 compose.yaml:60
