  1. python3
  用 Python 3 解释器执行后面的内容。

  2. -m agent_runtime
  不是直接跑某个 .py 文件，而是把 agent_runtime 当成一个模块/包来启动。
  这里会先进入 main.py:1，再调用 demo.py:61 里的 main()。

  等价理解接近于：

  python3 agent_runtime/demo.py ...

  但 -m 是更标准的包启动方式。

  3. "帮我分析这个 Dify CRM 客户查询工作流的 DSL，并告诉我该改哪些节点"
  这是位置参数 query，也就是“用户请求”。
  在参数定义里它对应：demo.py:19

  程序会拿这句话去做 skill 路由：

  - 看有没有显式 $skill
  - 看有没有命中 skill.json 里的 trigger
  - 看 token overlap

  路由逻辑在 router.py:22。

  4. --target-file Test_SearchCustomer.yml
  这是一个可重复参数，表示把 Test_SearchCustomer.yml 记入本次任务的运行时状态。
  参数定义在 demo.py:40。

  它当前的作用主要是：

  - 写进 RuntimeState.target_files
  - 再被注入到最终的 developer prompt 里

  数据结构在 models.py:33，注入逻辑在 prompt_assembler.py:67。

  这条命令实际会做什么

  在这个仓库里，完整流程是：

  1. 扫描 skills/ 下的 skill
     代码：skill_registry.py:56
  2. 用你的 query 给每个 skill 打分并选最高分
     代码：router.py:26
  3. 生成一个 task_id，创建运行时状态
     代码：demo.py:83
  4. 把状态保存到 .agent_runtime/state/<task_id>.json
     代码：runtime_store.py:21
  5. 把 skill 内容 + runtime state 组装成：
      - system
      - developer
      - user
        代码：prompt_assembler.py:8
  6. 最后打印一段 JSON，里面包括：
      - 是否命中 skill
      - 命中的 skill 名
      - 打分和原因
      - state 文件路径
      - 组装好的 prompt

  结合你这条命令的语义

  它的意思其实就是：

  “启动本地 agent_runtime，把这句中文需求当成用户请求，尝试路由到最合适的 skill，并把 Test_SearchCustomer.yml 作为当前任务的目标文件记录进运行时状态，最后输出可直接喂给大模型的 prompt JSON。”
