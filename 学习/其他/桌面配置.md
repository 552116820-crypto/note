# 几个常见的坑

**光猫桥接问题**
运营商光猫默认是路由模式，自己再接路由就是双 NAT，会影响 NAS/远程开发。
- 解决：打 10000 号要超管密码，把光猫改桥接，让自己路由 PPPoE 拨号
- 或者：直接用光猫做主路由，桌端只挂交换机/AP——简单但损失功能

**带宽匹配**
办的是 500M 宽带就别幻想跑 2.5G，光猫和入户线是瓶颈。先确认套餐：
- 千兆套餐 = CAT6 + 千兆路由器够
- 2000M / 万兆套餐 = 需要 2.5G/10G 网口和对应交换机

**房东和邻居**
- 改光猫位置 / 钻墙 → 必须先问房东，违约金不便宜
- 同小区蹭网风险高，路由 WPA3 + 隐藏 SSID

**退租复原**
- 3M 胶卡用吹风机热风吹 10 秒就能撕下来
- 残胶用酒精+银行卡刮，5 分钟搞定
- PVC 线槽是自粘的，同样吹风机处理

# 三、配置流程（5 步）

### 1. 改光猫为桥接模式
运营商光猫默认是路由模式，会导致**双 NAT**——影响远程开发、NAS 内网穿透、Tailscale 等场景。

打 10000 号（电信）/ 10010 号（联通）/ 10086 号（移动），要"超级管理员密码"，理由说"自己要换路由器拨号"。客服一般会给。

拿到密码后登录光猫 `192.168.1.1`（或 192.168.0.1）：
- 找到 WAN 配置 → 把当前的 INTERNET 连接改成 **Bridge（桥接）模式**
- 保存

光猫现在只做"光转电"，不拨号、不分配 IP。

### 2. GT-BE98 Pro 拨号
光猫 LAN 口 → 网线 → 路由器的 **10G WAN 口**。

浏览器进 `router.asus.com` 或 `192.168.50.1`：
- 一键设置 → 选 PPPoE
- 输入宽带账号密码（运营商给的，没有可以问客服要）
- 拨号成功后路由器会拿到公网 IPv4（如果运营商给的话）和 IPv6

### 3. 启用 2.5G 端口
华硕固件默认所有 LAN 都启用，2.5G 端口和千兆端口会自动协商速率——主机插哪个口都行。
- 进路由器后台 → LAN → 内网 IP，确认 DHCP 启用即可
- 进 系统 → 网络工具，看链接速率，2.5G 端口插了 2.5G 设备会显示 2500 Mbps

### 4. 桌端跑长线
路由器某个 2.5G LAN 口 → CAT6 网线 → 桌端：
- 设备少（1-2 台）：直接插主机
- 设备多（主机+NAS+显示器+笔记本网卡）：桌端加一个 **5 口 2.5G 交换机**，**TP-Link TL-SG105-M2** 或 **华硕 XG-U2008**，¥300-500

### 5. 验证速度
- 内网测速：`iperf3 -s` 在 NAS / 另一台机上跑，`iperf3 -c 对方IP` 在桌机上测——应该跑到 2.3-2.4 Gbps
- 外网测速：speedtest.cn 看 WAN 拨号速度

# 四、关键认知：2.5G 主要给内网，不是外网

这是很多人配完发现"没快"的核心原因。

| 场景 | 千兆够吗 | 2.5G 真有用 |
|---|---|---|
| 上网/下载 | 套餐 ≤1000M 都够 | 套餐 ≥2000M 才有意义 |
| NAS 看 4K 影片 | 够 | 多人并发或多客户端时有用 |
| NAS 备份代码/数据集 | 够慢 | **明显快**（千兆 110MB/s → 2.5G 280MB/s） |
| 远程 GPU SSH 开发 | 够 | 无感（瓶颈在外网延迟） |
| 训练数据集拷贝 | 慢 | **明显快** |
| WiFi 7 手机连本机 | 够 | 路由器内部转发，2.5G 是上限 |
| 2 台机器 P2P 传文件 | 100 MB/s | **280 MB/s** |

**核心价值**：内网传 NAS / 模型权重 / 数据集 / git 大仓库时，体感差异巨大。如果你只用来上网，千兆路由就够，省下的钱买 NAS。

# 六、几个隐藏的坑

**电信运营商 PPPoE 绑定**：少数地区运营商对 MAC 地址绑定严格，把光猫拨号迁到路由器后第一次连不上——需要"重置上网认证"或路由器克隆光猫 MAC。

5. **重直播的话上双线**：游戏走主光纤，OBS 推流走 4G/5G CPE 副线

# 四、"魔改"路由器要避雷

某些卖家会刷固件解锁发射功率到 30dBm（1W），号称"穿墙王"。这有三个问题：

1. **违法**——国家无线电管理规定家用 WiFi 功率 ≤20dBm，超了可被罚款
2. **干扰邻居**——你的信号变强，邻居的 WiFi 就被你压制，邻居反过来加功率，恶性循环
3. **路由器寿命短**——长期超功率运行 PA 容易烧

合法范围内提升信号的做法都比"魔改"靠谱：
- 路由器**放高**（书柜顶 / 墙挂）
- 路由器**放屋子中心**，不要塞墙角和金属柜里
- 天线**一半垂直、一半 45°**（4 根天线时）
- 卧室加一个 AP 做 **AiMesh** 节点（GT-BE88U 直接支持），有线回程效果最好
- 5GHz 设 **160MHz 频宽**、6GHz 设 **320MHz**（WiFi 7 客户端才用得上）

# 四、影响速度的真正瓶颈（按影响排序）

### 1. 硬盘类型（最大变量）

| 硬盘 | 写入速度 | 影响 |
|---|---|---|
| **NVMe SSD**（PCIe 3.0+） | 1500-7000 MB/s | 完全不是瓶颈 |
| **SATA SSD** | 500 MB/s | 不是瓶颈 |
| **HDD 机械盘** | **100-150 MB/s** | **千兆能跑满，但小文件多时拖慢明显** |

**千兆套餐配 HDD 是错配**——大型现代游戏（尤其 Mod/DLC 多的）小文件多，HDD 实际能用速度可能只有 30-50 MB/s。**装游戏一定用 SSD**。

### 2. Steam 下载区域设置

Steam → 设置 → 下载 → **下载区域**：
- **上海 / 广州 / 香港**：国内电信用户最快
- **首尔 / 东京**：联通用户有时更快（看跨网情况）
- **新加坡**：移动用户备选
- 不要选 "Auto"——Steam 自动选不一定最优

试 2-3 个区域，看哪个跑速度最高，固定那个。

### 3. 网络环境

| 因素 | 影响 |
|---|---|
| 有线 / WiFi | 有线稳定满速；WiFi 6/7 千兆能跑，但 PC 距离路由远会衰减 |
| 路由器拨号 vs 光猫拨号 | 光猫拨号双 NAT 可能掉 5-10% 速度 |
| 同网络其他流量 | 别人在看 4K Netflix / 直播会抢带宽 |
| 时间段 | 凌晨 2-6 点最快，晚 8-10 点最慢 |

### 4. 上行抢占下行（直播 + 下载的坑）

OBS 推流时下载会变慢——TCP ACK 包被上行挤压，下载吞吐量打折。
**直播时别同时下游戏**。

# 五、怎么让 Steam 跑满千兆

按效果排序：

1. **装 NVMe SSD**——三星 990 Pro / 西数 SN850X，1TB 约 ¥500
2. **PC 有线接路由器**（CAT6 千兆口）
3. **Steam 下载区域选上海/广州**
4. **Steam → 设置 → 限制带宽 → 选"无限制"**（默认就是，别改）
5. **关掉其他占带宽的程序**（迅雷、网盘自动同步、B 站直播伴侣）
6. **路由器 QoS 给 Steam 进程高优先级**（华硕 Adaptive QoS 里能直接选）
