## 项目架构

- 前端：Vue (VueAdmin 目录)
- 后端：ThinkPHP (app 目录)
- 前后端分离架构

## 前端数据从哪来

前端数据是通过**后端接口**返回的，不是直接连接数据库。

整个数据流：

```
Vue 前端页面 → axios HTTP 请求 → ThinkPHP 后端 API → 数据库 → 返回 JSON
```

### 1. 前端 API 层 (`VueAdmin/src/api/`)

每个模块定义了 API 请求函数，例如 `customer.js` 里的 `getCustomerList()` 调用 `/Customer/customerList` 接口。

```js
import request from '@/utils/request'

export function getCustomerList(query) {
  return request({
    url: '/Customer/customerList',
    method: 'get',
    params: query
  })
}
```

### 2. 请求封装 (`VueAdmin/src/utils/request.js`)

基于 axios 创建统一的 HTTP 请求实例：
- baseURL 为 `/adminApi`
- 请求时自动携带 `X-Token` 认证令牌

### 3. 后端路由 (`app/adminApi/`)

ThinkPHP 后端接收 `/adminApi/...` 路由，由 `controller/` 下的控制器处理业务逻辑，通过 `app/model/` 层访问数据库，最终返回 JSON 数据给前端。

## 页面加载数据的流程

每次进入一个新页面，Vue 组件会在生命周期钩子（通常是 `created` 或 `mounted`）中调用 API 函数，向后端发请求，拿到数据后赋值给组件的 `data`，页面就渲染出来了。

```
用户点击菜单 → Vue Router 跳转 → 组件加载 → created()/mounted() 触发 → 调用 api/xxx.js → axios 发 HTTP 请求 → 后端返回 JSON → 前端拿到数据绑定到页面
```

## 如何查找某个页面的数据来源

1. 在 `VueAdmin/src/api/` 下找到对应的 API 文件，查看调用了哪个接口
2. 在 `app/adminApi/controller/` 下找到对应的控制器，查看它查询了哪些表和字段
