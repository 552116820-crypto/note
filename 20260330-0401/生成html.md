## 用什么工具复刻

### 推荐方式：Claude 对话（最佳）

1. **Claude.ai 网页版**（claude.ai）— 直接对话生成
2. **Claude Code**（CLI工具）— 适合迭代修改
3. **Claude API** — 批量生成

### 为什么适合用 Claude

- 单文件 HTML，无需构建工具
- 纯 CSS 动画 + 原生 JS，不依赖框架
- 设计感来自精心的 CSS（渐变、阴影、圆角、动画）
- 内容和布局耦合在一起，适合 LLM 一次性生成

## 关键技巧总结

| 技巧 | 说明 |
|------|------|
| **单文件约束** | 明确要求"纯HTML单文件，不引入外部库"，避免生成需要构建的项目 |
| **CSS变量** | 要求用 `:root` 定义颜色变量，方便后续换主题色 |
| **数据驱动** | 要求将业务数据定义为JS对象，HTML通过JS动态渲染（如第二个文件的做法），方便后续改数据 |
| **分步迭代** | 先让Claude生成基础框架，再逐步加功能，不要一次要求太多 |
| **提供参考** | 可以把现有HTML文件内容粘贴给Claude说"参考这个风格，帮我做一个新主题的" |

## 如果没有参考HTML文件，只有MD文档怎么办

### 分两步走

#### 第一步：让 Claude 读你的 MD 文档

```
请阅读这个项目文档（粘贴MD内容或给文件路径），
理解其中的核心模块、技术架构、业务流程。
先不要写代码，列出你理解到的关键信息结构。
```

这一步让 Claude 先消化内容，避免遗漏。

#### 第二步：给出明确的可视化要求

关键是**说清楚5件事**：

```
基于上面的文档内容，帮我生成一个单文件HTML可视化页面。

1.【技术约束】
   纯HTML+CSS+JS单文件，不引入任何外部库/CDN，
   直接双击浏览器打开就能用。

2.【视觉风格】
   专业商务数据看板风格（不是普通文档页面），参考特征：
   - 顶部Hero区用渐变色背景+白色大标题+关键数字统计
   - 卡片式布局，每张卡片有圆角、阴影、hover上浮动效果
   - 用CSS变量管理主题色，整体配色协调
   - 背景色 #FAFAFA，字体用 PingFang SC / Microsoft YaHei

3.【布局结构】（选一种告诉它）

   A) 时间线/旅程风格：
      中间一条渐变竖线，节点左右交替排列，
      每个节点有圆形徽章+可展开的详情卡片

   B) 模块看板风格：
      可展开/折叠的模块卡片，左侧彩色边框，
      头部有筛选按钮栏，底部有统计图表

   C) 仪表盘风格：
      统计数字卡片 + 网格模块卡片 + 流程图

4.【交互效果】
   - 卡片可点击展开/折叠
   - hover有上浮+阴影加深动画
   - 如果有环形/流程图，加一个缓慢旋转动画
   - 移动端响应式适配

5.【内容要求】
   中文，数据从文档中提取，每个模块要包含：
   [根据你的需要指定，比如：名称、描述、负责人、
   技术栈、关键文件、痛点/方案/价值 等]
```

### 一个可以直接用的完整提示词模板

把下面这段话复制到 Claude 对话里，替换 `[...]` 的部分：

```
我有一份项目文档（见下方），请帮我做成一个
可视化HTML页面。

技术要求：纯HTML+CSS+JS单文件，零外部依赖，
浏览器直接打开。

视觉要求：专业商务数据看板风格，不是普通文档网页。
要有：渐变Hero顶栏+数字统计、卡片式布局带阴影和
hover动效、CSS变量管色彩、圆角+渐变+动画，
整体要有设计感，像一个产品发布页而不是技术文档。

布局：[时间线旅程 / 模块看板 / 仪表盘]，
包含 [筛选器 / 展开折叠 / 进度条 / 环形图] 等交互。

配色：[蓝色系 / 绿色系 / 自由发挥]，背景 #FAFAFA。

响应式：适配手机端。

以下是文档内容：
---
[粘贴你的MD文档]
---
```

### 有参考文件 vs 无参考文件的区别

| | 无参考文件 | 有参考文件 |
|---|---|---|
| 风格精度 | 靠文字描述，80%接近 | 直接对标，95%接近 |
| 提示词长度 | 需要详细描述视觉风格 | 只需说"参考这个风格" |
| 迭代次数 | 可能要2-3轮微调 | 通常1轮就到位 |

**最高效的做法**：如果你见过某个喜欢的HTML可视化页面，保存下来当参考文件。以后每次只需要说"参考这个风格，用我的新内容生成"就行了。
