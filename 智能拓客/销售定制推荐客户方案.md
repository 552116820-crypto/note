  # 销售定制推荐客户方案

  ## Summary

  目标是给每个销售生成一批“附近、像历史成交客户、质量较高”的公海客户推荐列表。
  推荐总分由三部分组成：客户评级分 + 历史成交匹配度 + 销售距离分，并支持销售或运营切换排序策略：距离优先、匹配度优先、
  客户评分优先。

  ## Key Changes

  - 客户基础池：只从公海客户中推荐，排除已分配、已成交、冻结、黑名单、无效联系方式、无可用地址/坐标的客户。
  - 客户评级分：沿用现有星级/总分，衡量客户自身质量，包括企业规模、信用风险、创新资质、市场信号、信息完整度。
  - 历史成交匹配度：用候选客户与历史成交客户在行业、主营产品、地区、规模、成立年限、风险、专利/资质等维度的相似度计算
    0-100 分。
  - 销售距离分：销售当前位置与客户经纬度计算距离，距离越近分越高；超过设定半径默认不推荐。
  - 可调权重：
      - 距离优先：距离 50% + 匹配度 30% + 客户评分 20%
      - 匹配度优先：匹配度 50% + 客户评分 30% + 距离 20%
      - 客户评分优先：客户评分 50% + 匹配度 30% + 距离 20%
      - 均衡模式：匹配度 40% + 客户评分 35% + 距离 25%

  ## Implementation Details

  - 需要补充的数据字段：公海状态、客户经纬度、坐标解析状态、历史成交标记、历史成交产品/金额/次数、销售当前位置。
  - 地址转坐标：用高德/腾讯/百度地图 API 批量把客户地址解析成 latitude、longitude，保存 geo_provider、geo_confidence、
    geo_status、geo_updated_at；查询推荐时不实时解析地址。
  - 匹配度计算：
      - 行业/主营标签/经营范围关键词：35%
      - 规模：注册资本、实缴资本、参保人数：20%
      - 地区/产业带：省市区：15%
      - 经营稳定性：成立年限、登记状态、年报年份：10%
      - 风险适配：司法、失信、处罚、限制高消费：10%
      - 能力信号：专利、商标、荣誉、补贴、中标、标准：10%
  - 距离分计算：
      - 0-3km：95-100 分
      - 3-10km：80-95 分
      - 10-30km：50-80 分
      - 30-50km：20-50 分
      - 超出半径：不推荐或低分展示
  - 推荐接口：POST /api/sales-recommendations/nearby-public-customers

    {
      "sales_id": "000455",
      "latitude": 31.2304,
      "longitude": 121.4737,
      "radius_km": 30,
      "mode": "balanced",
      "limit": 30,
      "min_star": 2,
      "industry_tags": ["纺织", "面料"]
    }
  - 响应结果：

    {
      "items": [
        {
          "customer_id": 12345,
          "customer_name": "某某纺织有限公司",
          "recommend_score": 82.4,
          "star": "3星",
          "customer_score": 68.5,
          "match_score": 86.0,
          "distance_km": 4.2,
          "distance_score": 91.0,
          "address": "江苏省苏州市吴江区...",
          "recommend_reason": "距离4.2公里，主营标签与历史成交客户相似，客户风险较低"
        }
      ]
    }

  ## Recommendation Flow

  - 离线任务：
      - 批量解析客户地址为经纬度。
      - 计算并刷新客户评级分。
      - 基于历史成交客户生成成交画像或相似度索引。
  - 实时查询：
      - 接收销售当前位置和排序模式。
      - 先按公海状态、权限、半径、星级做过滤。
      - 计算距离分、匹配度、客户评分。
  - 领取后处理：客户被领取或分配后，从公海推荐池移除；若销售忽略/拉黑/无效，应记录反馈并降低后续推荐权重。

  - AI 大模型可以用在三个地方：从经营范围/企业简介中抽取主营产品标签；生成“为什么推荐这个客户”的自然语言解释；基于历史
    成交客户总结销售偏好的行业/区域/客户类型。
  - 不建议让大模型直接决定最终排序。最终排序应由可解释的规则分数控制，大模型做标签增强和推荐理由生成更稳妥。

  ## Test Plan

  - 验证只推荐公海客户，不返回已分配、已成交、冻结、黑名单客户。
  - 验证不同排序模式下，同一批客户排序会按权重变化。
  - 验证距离边界，半径外客户不进入结果。
  - 验证高风险客户即使距离近、匹配度高，也会被降权或过滤。
  - 验证历史成交客户画像变化后，匹配度和推荐结果会更新。
  - 验证销售领取、忽略、无效反馈会影响后续推荐。

  ## Assumptions

  - 销售位置由前端实时上传经纬度。
  - 公海客户地址可以通过地图 API 批量补齐经纬度。
  - 历史成交客户至少能标记出“哪些客户成交过”，最好还能补充成交产品、成交金额、成交次数。
  - 默认推荐模式为 balanced，默认半径 30km，默认返回 30 条。
