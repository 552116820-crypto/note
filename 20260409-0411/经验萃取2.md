## 第一部分：AI落地的误区与破局之道

### 1.1 数字化转型面临的深度变化

当前企业在推进AI落地时面临三层核心挑战：

| 层面 | 核心问题 | 本质 |
|------|---------|------|
| **业务层：复杂性提升** | AI到底能做多少？经验是否可复制？ | 能力问题 |
| **IT层：信任被挑战** | 业务是否真正认可？是否愿意使用？ | 可信问题 |
| **决策层：链路不可控** | AI决策是否可靠？是否可审计、可追溯？ | 落地问题 |

### 1.2 场景选择，决定了AI落地的成败

**适合AI解决的场景特征：**
- 业务目标清晰，结果可验证
- 流程明确，可拆解为多个步骤
- AI作为辅助工具，而非最终决策者

**场景优先级模型（同心圆模型）：**
- 核心圈（可立刻自动化）：高重复 + 高可量化
- 中间圈（ROI最高）：规则清晰 + 低损失
- 外圈（长期AI替代方向）：低人工依赖

### 1.3 AI落地不是"独角戏"

AI落地是上下协同、分工联动的**组织接力**，需要四个层级协作：

```
              高层领导
        明确总体方向与重点领域，统筹资源与推进机制
              ↙        ↘
      业务中层            信息化部门
  明确业务边界与应用范围    提供技术底座、数据治理
  提出需求与验收标准       系统集成与运行保障
  组织试点推广
              ↘        ↙
          基层一线与业务骨干
    按规范使用、反馈问题、参与优化
        推动形成常态化应用
```

### 1.4 AI落地的关键：构建可控的治理体系

通过原则、机制与评价，实现方向、过程与结果的可控：

| 维度 | 内容 |
|------|------|
| **可控原则** | 明确AI应用边界与基本约束 |
| **治理机制** | 建立跨部门协同与推进机制 |
| **评价导向** | 以业务价值与可复用性为核心指标 |

### 1.5 AI落地的最佳分工：80%自动化 + 20%人类决策

| 人工角色（20%审核） | AI角色（80%模型完成） |
|---|---|
| 高价值判断（风险/决策） | 高重复任务 |
| 结果背书（责任承担） | 标准化流程 |
| 异常处理（非标准情况） | 成本敏感环节 |

### 1.6 AI做不成能力，只会停留在工具层面

要让AI真正成为组织能力，需要三个支撑形成闭环：

- **能力框架**：让能力可复用
- **资源与协同**：让事情能推进
- **路线节奏**：让价值可放大

### 1.7 AI落地三阶段演进

| 阶段 | 目标 | 关键动作 |
|------|------|---------|
| **准备阶段** | 让AI"能用" | 业务梳理（明确场景边界）、数据准备（保证输入质量）、规则建模（固化关键逻辑）、人员支撑（明确人机分工） |
| **落地阶段** | 让AI"可控" | 小范围快速上线（试点）、人机协同运行（不直接替代）、持续纠错优化（反馈闭环）、过程可记录（可复现） |
| **运营阶段** | 让AI"有价值" | 流程嵌入（进入正式业务）、评价聚焦（指标统一）、结果导向（关注产出）、长期运营（持续迭代） |

> **核心观点：** AI落地不是一次上线，而是通过"经验萃取+技术落地"，实现"可用 → 可控 → 有价值"的持续演进过程。

### 1.8 系统定位升级：从"固定模板机器取数"到"自然语言驱动的智能分析"

| 维度 | 传统模式（展现写死、数据但不灵活） | 智能体（理解意图、灵活准确） |
|------|---|---|
| 交互方式 | 问法有限 | 任意询问 |
| 迭代能力 | 迭代困难 | 快速生成 |
| 透明度 | 口径不透 | 口径透明 |
| 深度分析 | 不能追问 | 连续追问 |

## 第二部分：实战路径 -- 五步构建企业智能体

### 总体框架：定-挖-萃-建-优

用五个生动的比喻串联整个流程：

| 步骤 | 比喻 | 核心动作 | 说明 |
|------|------|---------|------|
| **定** | 选战场 | 锚定价值场景 | 圈地盘、找机会 |
| **挖** | 立靶子 | 挖痛点根因 | 拆痛点、定靶心 |
| **萃** | 捞干货+挤水分 | 经验萃取 | 破幻觉、拆保守 |
| **建** | 搭积木 | 建智能体 | 配人机、落应用 |
| **优** | 养孩子 | 优化迭代 | 看数据、持续长 |

### 2.1 第一步：定 -- 锚定价值场景

**高价值正向机会筛选标准：**

| 维度 | 先不做的（排除项） | 首选的 |
|------|---|---|
| 具体内容 | 设备周期性维护、突发应急维修、业主个性整改（如插座位置等） | 质量巡检 -- 质量风险识别 |
| 不选/选择原因 | 低重复、不可量化、高人工依赖 | 高频发生、返工成本高、经验可复制 |
| 成果 | -- | 准确率提升50%，返工降低30% |

### 2.2 第二步：挖 -- 挖痛点根因，界定协同机会

- **核心目标**：明确"做什么"，将模糊痛点转化为可落地的智能体开发主题
- **分析方法**：运用痛点根因分析（阿波罗根因分析）和机会面布罗盘进行深入剖析
- **关键产出**：产出《负向高价值机会场景清单》

**分析流程：** 岗位目标 → 工作任务/流程 → 工作场景/痛点 → 正向机会查找 + 痛点根因分析 → 正向机会和负向机会比对 → 高价值清单

**案例 -- 质量巡检场景的根因分析：** 访谈靶心清单围绕四层漏斗展开：
1. 人太少
2. 跑不到
3. 看不见
4. 看不准

核心访谈问题：
- 老师傅如何判断"哪些区域/时段风险最高"？
- 老师傅通过什么"远程迹象"推断内部问题？
- 老师傅的"直觉"和"手感"是什么帮助看见？

### 2.3 第三步：萃 -- 萃专家经验，构建数字空间

**访谈方法（漏斗模型）：** 访谈开始 → 目标聚焦 → 场景回放 → 提炼步骤 → 追问细节 → 深挖难点 → 复述确认 → 表示感谢 → 访谈结束

**四维对话框架：**

| 维度 | 关注点 |
|------|--------|
| **技术（Technology）** | 技术成熟度、技术可扩展、数据/债务 |
| **用户（User）** | 痛点/体验、用户参与度、场景兼容性 |
| **商务（Business）** | ROI与市场、竞争力定价、生态协同 |
| **风险（Risk）** | 合规性流程、颠覆性依赖、长尾风险 |

> **金句：** "访谈是挖矿，研讨是炼金。只挖不炼，矿石变不成金子。"

**案例 -- 质量工程师岗位机会清单：**

| 二/三级流程 | 机会主题 | 实际流程（做法） | 希望AI能做的 | 过程使用材料和工具 | 输出物 |
|---|---|---|---|---|---|
| 质量巡检-风险识别 | 如何在每人2万平的管理面积里判断"哪些区域/时段风险最高"？ | 1.功能属性（水电燃气、重负荷功能区）2.历史记录（出毛病的地方）3.环境促退（室外、半户外、朝北角）4.人群密度5.隐蔽程度6.特殊时间安排 | 希望AI能最快提供风险排序 | 建筑竣工平面图、维修工单记录表、温湿度/照度仪、巡检记录清单等 | 风险识别地图、设备健康档案、预防性维护计划 |

### 2.4 第四步：建 -- 建智能体（搭积木，配人机，落应用）

将萃取的专家经验二次筛选价值经验，构建智能体。

### 2.5 第五步：优 -- 优迭代闭环，持续释放价值

| 维度 | 内容 |
|------|------|
| **核心目标** | 明确"怎么持续做"，让智能体与业务共同进化，确保长期价值的持续释放 |
| **实施方法** | 数据驱动（基于监控数据优化）、用户反馈（收集真实使用体验）、专家评审（定期评估性能指标）、CI/CD（自动化持续集成部署） |
| **关键产出** | 《智能体迭代优化报告》、《企业智能体资产库》 |

### 2.6 项目成果

| 指标 | 数据 |
|------|------|
| 覆盖岗位 | **15个** |
| 场景数 | **1000+个** |
| 智能体数 | **700+个** |
| 调用次数 | **100W+次** |

## 第三部分：SBP即时萃取三步法（核心工具）

SBP是本次大会重点介绍的经验萃取工具，形成一个三角模型：

```
            ▲
           / \
          / B \      Balanced Review -- 理性复盘
         /     \     按照结构化的逻辑框架梳理出案例的
        /-------\    脉络，有背景有故事，有实践有结
       /    A    \   果，有对标有反思，这样的萃取结
      /___________\  果才丰满。
     S            P

S = Select Case        P = Perceptual Encapsulation
    选择案例                感性封装
    选择有价值的案例，        案例萃取成果的封装不是
    理性检验后感性            一次性入库，而是案例萃取
    价值最具的案例才有         成果与应用者之间持续
    更大的萃取价值和           互动的动态记录，鲜活的
    传播力。                  案例自带转化力。
```

### 3.1 第一步：Select Case -- 选择案例

选择一个 **Outstanding CASE**，需要同时满足理性和感性两个维度的评价标准：

**理性评价维度：**
- **富有成效**：事件取得了成功，有方法可以总结
- **具体聚焦**：边界清晰，内容明确，避免空泛模糊。小主题比大主题更好

**感性评价维度：**
- **感性喜悦**：有发自内心的成就感、认同感，包含感性因素
- **具有挑战**：有难度有冲突，不是简单可以完成

> 评分说明：以上两项每项10分，总分20分，要求不低于16分。

### 3.2 第二步：Balanced Review -- 理性复盘

采用拼图式的六维结构化框架对案例进行理性复盘：

| 模块 | 英文 | 核心问题 |
|------|------|---------|
| **背景** | Situation | 发生了什么事？时间、地点、人物...... |
| **任务** | Task | 面临什么问题，要完成什么任务？ |
| **行动** | Action | 采取了哪些行动？ |
| **结果** | Result | 取得了什么结果？ |
| **对标** | Benchmark | 对标的对象是谁，对标带领什么启发？ |
| **反思** | Review | 有什么经验和教训？ |

### 3.3 第三步：Perceptual Encapsulation -- 感性封装

感性封装是让案例"活起来"的关键步骤，从四个维度进行深度加工：

| 维度 | 英文 | 核心问题 |
|------|------|---------|
| **信息** | Objective | 有哪些信息、印象最深刻？ |
| **感受** | Reflective | 有哪些感受最强烈？ |
| **启发** | Interpretive | 有哪些最有价值的规律？ |
| **行动** | Decision | 由此引发你哪些最要紧的行动？【后续跟踪记录】行动结果及反思 |

> **关键洞察：** 案例萃取成果的封装不是一次性入库，而是案例的萃取成果与应用者之间持续互动的动态记录。鲜活的案例自带转化力。

## 第四部分：复盘+萃取方法论

### 4.1 理论基础：大卫·库伯经验学习圈

复盘+萃取的底层理逻辑基于大卫·库伯（David Kolb）的经验学习圈：

```
    积极实践  ←————————→  产生经验
   规律指导实践             从实践中获得成功
   提升组织效率             或失败的经验
        ↑                      ↓
    抽象概念化  ←————————→  反思性观察
   萃取经验                  定期复盘
   高位经验                  分析成败
   形成规律                  解决问题
                             叙述经验
```

> **核心观点：**
> - "不盘不萃不会"
> - 任正非：组织最大的浪费，就是经验的浪费！！
> - AACTP：没有战萃一体的战役，就是犯罪！！

### 4.2 必赢之战中的"复盘+萃取"时机

**边战边萃、战萃一体、战传合一**

| 必赢之战阶段 | 项目初起 | 步及正轨 | 项目至中 | 项目中后 | 项目结项 |
|---|---|---|---|---|---|
| **复盘** | 盘目标 | 盘策略 | 盘问题 | 盘经验 | 盘规律 |
| **萃取** | 战前研究标杆、巧做迁移 | 验证 | 战中经验涌现、美联如快速复制 | 验证 | 战后固化打法、迭代系统 |

- **复盘原则**：大事阶段盘，关键突发事件马上盘，项目结项全面盘
- **萃取原则**：有经验必萃取，有萃取必验证，验证后必固化，固化后必迭代

### 4.3 必赢之法杀手锏 -- 复盘+萃取

通过漏斗模型层层聚焦：**战场 → 任务 → 战斗过程 → 亮点/难点**，再通过AACTP项目复盘画布进行系统思考、全面扫描，找到卡点和萃取点。

**萃取点筛选维度：**

| 维度 | 标准 |
|------|------|
| 价值 | 低价值 → 高价值 |
| 难度 | 低难度 → 高难度 |
| 紧急重要 | 不紧急/不重要 → 紧急/重要 |
| 覆盖面 | 覆盖小/低频 → 覆盖广/高频 |

> 萃取主题描述格式：《如何XXXXXXXXXXXX？》

### 4.4 复盘 -- 回顾战斗过程，扫描萃取点

复盘的核心是找到两类萃取点：
- **普人卡点、难点、失控点** → 就是萃取点
- **牛人亮点、掌控点** → 就是萃取点

### 4.5 AACTP项目复盘画布（Project Review Canvas）

这是一个结构化的复盘工具，分为四大板块：

| 板块 | 颜色 | 核心内容 |
|------|------|---------|
| **回顾目标** | 红色 | 战略解码 -- 目标分解、对应行动与结果 |
| **评估策略** | 橙色 | 战术支撑 -- 对行动进行目标评估 |
| **反思过程** | 蓝色+绿色 | 战斗反思 -- 好的做法（结果与影响）、不足的做法（结果与影响+改善做法）、变化趋势（延续与影响+驱动因素） |
| **总结规律** | 深绿色 | 战备系统 -- 结构、规律、行为 → 行动计划（继续行动、停止行动、开始行动） |

> 口号："让赢清清楚楚，让输明明白白"

### 4.6 复盘+萃取应用场景

| 场景 | 具体应用 |
|------|---------|
| **01 必赢之战** | 战略取舍、举全员之力、生死攸关的战役。案例：潮玩能量（潮玩年营收破亿）、Realme等 |
| **02 行动学习** | 战略落地、绩效提升、干部培养。案例：广新集团80/85/90新青年培养 |
| **03 重大项目** | 年度前三的重点工作。案例：深中通道工程等 |
| **04 年中/年终** | 半年度、年度总结，计划制定时。案例：国药、字节跳动、京东、松下电器、广汽丰田等 |
| **05 创新探索** | 新业务/新市场，第二增长曲线。案例：出海--机器人犬场交易、国产化--某企业大农业国际TOG等 |

### 4.7 两种萃取类型对比

| 类型 | 基于人才培养的经验萃取 | 必赢之战中的复盘+萃取 |
|------|---|---|
| **特征** | 静态能力素质 | 动态作战赋能 |
| **核心** | 围绕能力、系统沉淀、体系输出、长期投入 | 贴近战场、边战边萃、战传合一、及时敏捷 |
| **产出** | 知识资产、百科全书 | 生死攸关、及时敏捷、锦囊妙计 |

## 第五部分：经验萃取核心技术与方法

### 5.1 透过视频把握经验萃取成功的关键要素

经验萃取的七大关键要素：

1. **萃取主题** -- 要找基于岗位关键任务的痛点课题（如：如何射击运动中的兔子并提高命中率？）
2. **经验专家** -- 有成功实践经验或间接经验的人（顺藤摸瓜）
3. **萃取师** -- 有把控方向、信息提炼、善沟通的人（翰林）
4. **萃取方式** -- 观察、访谈
5. **传播载体** -- 口号、模型、案例、课程
6. **传播方式** -- 翰林讲课、面授、考试等
7. **复制应用** -- 每天晨练练、训练、复述记忆、实战应用、从知识到行为转变

### 5.2 淘宝原则（深、透、准）

经验萃取研讨的核心原则，形成"深→透→准"的循环：

1. 对照我的**难痛重点**，创建打破沙锅场域
2. 看似相同中找不同，推翻阻断**路径依赖**
3. 不仅挖掘显性行动，更要探询**动机假设**
4. 在偶然中找确定性，在多场景**举一反三**
5. 如遇无解疑难杂症，搅拌群智**多元共创**

**价值认同：** 经验即资产，传承即责任。经验不仅是个人财富，更是组织宝贵的"知识资产"。传承不仅是责任，更是扩大个人影响力的有效方式。

三种产出形式：

### 6.4 师课同建实战：从教材思维到用户思维的课件打磨

**三大误区：**
- 误区一：听完课程开发课，就能开发出好课程
- 误区二：课件写完，课程就完成了
- 误区三：课程好不好，培训师自己说了算

**核心公式：**
> **好课程 = 初稿 x （辅导 + 优化 + 试讲 + 反馈）^n**

- 每一轮打磨，课程质量就提升一个台阶
- 课程好不好，学员和业务专家说了算

### 7.1 SECI模型（组织知识转化与创造的过程）
