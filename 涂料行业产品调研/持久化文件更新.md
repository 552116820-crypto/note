## "等到第二次复用再写 template" 是什么意思

**Template 在这项目里特指**：`.claude/skills/research-one-company/templates/template_*.py` 里的可执行骨架。subagent 调研一家新公司时，主会话告诉它"这家是 X 类站点，复制 template_X.py 改 TODO 占位即可跑"，省去 subagent 从零写抓取代码。

现在已有 5 个：`static_aspx`（静态 ASP）/ `wordpress_dt`（WordPress + DataTables）/ `spa_api`（SPA + webpack API）/ `image_codes`（凡科建站图里 SKU）/ `jz_listing`（JZ 建站）。

**"第二次复用门槛"指：**

| 阶段 | 做什么 | 不做什么 |
|---|---|---|
| 第 1 家某类型站点 | 写一次性脚本 `scripts/crawl_X.py` 搞定它，把站点踩坑记到 memory | **不抽 template** |
| 第 2 家同类型站点 | 发现 90% 处理流程一样、改几个 URL/正则就能复用 → 此时**才**沉淀 `template_X.py` | — |

**为什么不在第 1 家时就抽 template：**

1. **没有数据点判共性 vs 个性** — 抽象需要 ≥2 个案例。一个案例做出来的 template 里满是只属于这家公司的硬编码，下次真遇到第 2 家时发现不适配，要么改 template、要么仍然另起炉灶 — 等于写两遍
2. **维护成本** — 每加一个 template 都要跟着 SKILL.md / scripts/lib 演进。一次性脚本可以放在 scripts/ 任其闲置；template 是"公共 API"，演进时容易遗漏
3. **过度抽象比重复代码更难修** — 重复代码读得懂、改得了；糟糕的抽象会把后来者带歪

**这次任务我违反这条的具体情况：**

我做完 cnscc + yipsink 后写了 `templates/template_pdf_manual.py`，看似"两家"算第二次复用 — 但实际：

- cnscc 是**图片型** PDF（扫描册 → 渲染 PNG → subagent OCR）
- yipsink 是**文字型** PDF（pdfplumber 抽文本 → 正则提 SKU）
- **处理路径完全不同**，不是同一个 template 的两次复用，而是同一次任务里发现两种独立变体
- 没有第 2 家图片型 PDF 公司验证 cnscc 那套，也没有第 2 家文字型 PDF 公司验证 yipsink 那套

正确做法（也是这次最终修复后的状态）：
