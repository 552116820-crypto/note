先纠正一个理解上的前提：**你并不是在 eth0 / eth1 / eth2 之间"选一条走"**。这不是三条备选路径,而是同一批网卡在不同启动下被随机改的名字。备忘里说的"网卡名不稳定"就是这个意思。

## eth0/eth1/eth2 是什么

`ethN` 就是 Linux 给网卡起的名字,`eth` = ethernet,后面的数字只是开机时的枚举序号。关键点:**在 WSL/Hyper-V 下这个枚举顺序不固定**——同一块虚拟网卡,这次重启叫 eth1,下次可能叫 eth0 或 eth2。所以 `eth0/1/2` 本身**没有语义**,不是"eth0 永远是内网、eth1 永远是隧道"。备忘里"同一天内实测过 eth0、eth1"、README 写死 eth2,都是这一个网卡在不同时刻的不同名字而已。

## 你真正要区分的是两块网卡(按功能,不按名字)

你这台机器上有两块有意义的网卡,该按 **IP / 路由** 认,不该按名字认:

| 功能 | 怎么认 | 现在(本次启动)是谁 |
|------|--------|-------------------|
| 普通内网 / 上网 | 带 **default 路由** 那块(`192.168.40.x`,网关 192.168.40.1) | **eth0** |
| 业务库代理隧道 | 带 **198.18.x** 地址那块(几乎所有公网 IP 段都 `via 198.18.0.2` 走它) | **eth1** |

连业务库 `amer_crm_v2`(47.114.77.122)的流量,从路由表看是被 `128.0.0.0/1`、`32.0.0.0/3` 这些大段全部导到 `198.18.0.2 dev eth1`,所以**今天走的是 eth1**。eth2 此刻根本不存在。

## 所以"什么时候走哪个"的正确问法

不是看名字,而是**每次先探测哪块网卡带 198.18.x**,那块就是隧道,业务库就走它:

```bash
ip -o -4 addr | grep 198.18    # 当前 → eth1
```

然后对那块设 MTU(备忘的核心动作):

```bash
sudo ip link set dev eth1 mtu 1400   # "eth1" 要换成上面探测出来的名字
```

我刚查了一下,**现在 eth1 的 MTU 已经是 1400**(不是默认黑洞的 9000),隧道这块已经是好的状态,可以直接跑连库的东西。`scripts/load_test.py` 里的 `tunnel_dev()` 做的就是上面这套自动探测,所以跑它不用手动指定网卡;**别信 README 里写死的 eth2**。

一句话总结:eth0/1/2 是会漂的网卡名,你要认的是"带 198.18.x 的那块=隧道",今天恰好叫 eth1。
