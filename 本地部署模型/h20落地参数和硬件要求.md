• 截至 2026年3月17日，单卡 H20 96GB 全卡 跑 Qwen/Qwen3.5-27B 没问题，但如果你要的是“上线后稳定、延迟别太难看”的配置，我建议按 纯文本服务、32K 上下文、中低并发 来配，不要把官方 262K 的示例当成单卡目标，因为官方
  那条 vllm serve 命令是 8 卡 --tensor-parallel-size 8。

  推荐启动参数

  vllm serve Qwen/Qwen3.5-27B \
    --host 0.0.0.0 \
    --port 8000 \
    --tensor-parallel-size 1 \
    --dtype bfloat16 \
    --gpu-memory-utilization 0.92 \
    --max-model-len 32768 \
    --max-num-seqs 8 \
    --max-num-batched-tokens 16384 \
    --reasoning-parser qwen3 \
    --default-chat-template-kwargs '{"enable_thinking": false}' \
    --language-model-only

  这套参数的意思很直接：

  - --tensor-parallel-size 1：单卡 H20。
  - --max-model-len 32768：单卡先从 32K 起步，别直接上 262K。
  - --default-chat-template-kwargs '{"enable_thinking": false}'：Qwen3/Qwen3.5 默认会 thinking，关掉后首 token 延迟和整体响应会更稳。
  - --language-model-only：如果你是企业文本助手/RAG/代码助手，建议开这个，能省掉视觉编码器占用。

  如果你们要 图像/视频输入，就删掉 --language-model-only，然后把 --max-model-len 先降到 16K，--max-num-seqs 先降到 4。这部分是我的工程建议，不是官方硬门槛。

  能流畅运行的硬件要求
  下面 CPU / RAM / 磁盘 是我基于官方权重大小和 vLLM 缓存机制给的工程建议：

  - GPU：1x H20 96GB 全卡，别用 48GB/24GB 这种 MIG 切片。
  - CPU：建议 32 vCPU，最低 16 vCPU。
  - 内存：建议 256GB RAM，最低 128GB RAM。
  - 磁盘：建议本地 NVMe 200GB+ 空闲；最低 120GB+。
  - 场景前提：纯文本、32K 上下文、中低并发。

  原因：

  - 官方 Qwen3.5-27B BF16 权重仓库大小大约 55.6GB。
  - 模型原生上下文是 262,144，但官方 262K 的 vLLM 示例是 8 GPU。
  - H20 官方文档里是 96GB 显存。

  如果你要 更稳、更宽松，上线时其实更建议把模型名换成 Qwen/Qwen3.5-27B-FP8。官方 FP8 仓库大小大约 30.9GB，同一张 H20 上会明显更轻松，尤其你想把上下文抬到 64K 或者把并发再往上拉。
