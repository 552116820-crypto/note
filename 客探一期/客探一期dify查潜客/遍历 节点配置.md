• 太好了，下面给你可直接落地的工作流方式：一家公司创建一篇知识库文档。

  流程

  - HTTP节点A：拿原始响应（含 body）
  - 代码节点B：用我给你的“30家企业版”脚本，输出 txt（多企业用空行分隔）   
  - 代码节点C：把 txt 拆成 docs 数组（每条含 name/text）
  - 迭代节点D：遍历 docs
  - HTTP节点E（放在迭代里）：调用“创建知识库文档 API”

  代码节点C（直接复制）

  def main(txt: str) -> dict:
      import re

      blocks = [x.strip() for x in (txt or "").split("\n\n") if x.strip()]
      docs = []

      for i, block in enumerate(blocks, 1):
          name = f"企业档案_{i}"
          m = re.search(r'"企业名称":"(.*?)"', block)
          if m and m.group(1).strip():
              name = f"{m.group(1).strip()}_企业档案"

          docs.append({
              "name": name,
              "text": block
          })
 
      return {
          "docs": docs
      }
 
  迭代节点D配置
 
  - 输入数组：代码节点C.docs
 
  HTTP节点E（迭代内）
 
  - Content-Type: application/json
  - Body 里把字段绑定到当前迭代项：
      - name -> 当前 item.name
      - text -> 当前 item.text
  - 其它必填（如 dataset_id）按你的 API 文档填写。
 
  关键点：不要手写模板变量，直接用 Dify 的变量选择器点选 item.name 和 item.text，可避免转义和截断问题。
