
  但你可以在工作流里做到“等效预设”：创建文档 → 立刻再调一个接口给该文档赋元数据。官方在“通过 API 维护知识库”的示例里给了文档元数据赋值接口（支持一次给一个或
  多个文档写入 metadata）。(docs.dify.ai (https://docs.dify.ai/en/use-dify/knowledge/manage-knowledge/maintain-dataset-via-api))

  ### 推荐做法（工作流里两步串起来）

  1)（一次性）给知识库新增/确保元数据字段存在：POST /v1/datasets/{dataset_id}/metadata，创建字段名和类型。(docs.dify.ai (https://docs.dify.ai/en/use-dify/
  knowledge/manage-knowledge/maintain-dataset-via-api))
  2) 创建文档：POST /datasets/{dataset_id}/document/create-by-text 或 .../create-by-file。(docs.dify.ai (https://docs.dify.ai/api-reference/documents/create-
  a-document-from-text))
  3) 给刚创建的文档赋元数据：POST /v1/datasets/{dataset_id}/documents/metadata，传 operation_data（里面带 document_id + metadata_list）。(docs.dify.ai
  (https://docs.dify.ai/en/use-dify/knowledge/manage-knowledge/maintain-dataset-via-api))
  4)（可选）用“获取文档详情”验证是否写入：该接口支持通过 metadata=only/all/without 控制返回哪些元数据。(docs.dify.ai (https://docs.dify.ai/api-reference/
  documents/get-document-detail?utm_source=openai))

  ### cURL 示例（你可以在工作流的 HTTP Request 节点里照搬）

  下面用 source=crm 做例子（把域名换成你的 Dify 实例地址也行）：

  # 1)（只需做一次）创建一个 metadata 字段：source
  curl -X POST 'https://api.dify.ai/v1/datasets/<dataset_id>/metadata' \
    -H 'Authorization: Bearer <api_key>' \
    -H 'Content-Type: application/json' \
    -d '{"type":"string","name":"source"}'

  # 2) 创建文档（以 text 为例）
  curl -X POST 'https://api.dify.ai/v1/datasets/<dataset_id>/document/create-by-text' \
    -H 'Authorization: Bearer <api_key>' \
    -H 'Content-Type: application/json' \
    -d '{
      "name":"doc1",
      "text":"...",
      "indexing_technique":"high_quality",
      "process_rule":{"mode":"automatic"}
    }'

  # 3) 给文档写入元数据（用上一步返回的 document.id + 第 1 步的 metadata_id）
  
metadata_id：验证
{

    "id": "7707845e-365e-4f8a-81f6-6942341c94f8",

    "type": "string",

    "name": "test"

}

  curl -X POST 'https://api.dify.ai/v1/datasets/<dataset_id>/documents/metadata' \
    -H 'Authorization: Bearer <api_key>' \
    -H 'Content-Type: application/json' \
    -d '{
      "operation_data":[
        {
          "document_id":"<document_id>",
          "metadata_list":[
            {"id":"<metadata_id>","name":"source","value":"crm"}
          ]
        }
      ]
    }'

  补充两点你在工作流里会踩到的点：

  - metadata_list 里的 value 需要和字段 type 匹配，否则会报 invalid_metadata 之类的错误。(docs.dify.ai (https://docs.dify.ai/en/use-dify/knowledge/manage-
    knowledge/maintain-dataset-via-api))
  - 创建文档接口在不同文档/版本里有 create-by-text vs create_by_text 这种命名差异；以你知识库页面 API ACCESS 里显示的实际路径为准（但“创建后再赋 metadata” 这
    个思路不变）。(docs.dify.ai (https://docs.dify.ai/api-reference/documents/create-a-document-from-text))
