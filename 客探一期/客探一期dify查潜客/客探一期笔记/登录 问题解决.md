• 不建议（也大概率不会生效）直接写成 ...?sales_id=000002。

  - 按 Dify 的嵌入规则，如果你是用“URL/iframe 方式”把 inputs 传进去，query 参数的值必须先 GZIP 压缩 → Base64 → 再 encodeURIComponent，不是明文直接传。
  - 明文 000002 通常会被当成普通 query string，Dify 不会把它解析成应用的 inputs.sales_id（所以你会看到应用仍然认为缺少必填输入）。
  你有两种正确做法，选其一：

  1. 你坚持用“链接/iframe”传参（需要编码）

  - 000002 按规则编码后是：H4sIAAAAAAAEADMwAAIjAMpkSNcGAAAA
  - 链接应写成（注意只有一个 ?，后续参数用 &）：
    http://dify.amer.cn/chat/qQrRtKvsKyEmBoho?sales_id=H4sIAAAAAAAEADMwAAIjAMpkSNcGAAAA (dify-6c0370d8.mintlify.app (https://dify-6c0370d8.mintlify.app/
    versions/3-0-x/en/user-guide/application-publishing/embedding-in-websites))
    （如果你编码后的字符串里出现 = / + / /，就必须再做 URL 编码；你这个值刚好不含这些字符。）

  2. 推荐：用 <script> 嵌入（不需要编码，直接传明文 inputs）
     这种方式可以直接在配置里写 inputs: { sales_id: '000002' }。
     同时你还可以设置 systemVariables.user_id（相当于 API 的 user），让“会话跟着销售走”，例如 sales:000002。

  示例（放在你们 CRM 工作台页面里）：

  <script>
    window.difyChatbotConfig = {
      token: 'qQrRtKvsKyEmBoho',
      baseUrl: 'http://dify.amer.cn',
      inputs: { sales_id: '000002' },
      systemVariables: { user_id: 'sales:000002' }
    }
  </script>
  <script src="http://dify.amer.cn/embed.min.js" id="qQrRtKvsKyEmBoho" defer></script>

  补一句重要的：无论你用 URL 还是 <script>，sales_id 都是“前端可见/可改”的参数，不要把它当作权限凭证；真正的数据权限要在你们 CRM 后端接口侧按企业微信登录态校
  验。
