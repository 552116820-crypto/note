要将 Dify 聊天框嵌入企业微信工作台的 CRM 系统，并实现 **自动识别当前用户 + 使用 `sales_id` 自动登录**，你可以按照以下步骤操作：

---

## ✅ **整体思路**

1. **在企业微信中嵌入一个 Web 应用页面**（承载 Dify 聊天界面）。
2. **通过企业微信 API 获取当前用户信息**（如 user_id）。
3. **将企业微信 user_id 映射到 CRM 中的 `sales_id`**（可通过后端接口实现）。
4. **使用 `sales_id` 自动登录 Dify**（通过 Dify API 或自定义登录机制）。

---

## 🛠️ **具体步骤**

### **1. 在企业微信中创建应用并嵌入页面**

- 在企业微信管理后台创建一个 **自建应用** 或 **网页应用**。
- 在应用设置中配置 **可信域名**（用于嵌入 Dify 页面的域名）。
- 在 CRM 系统中添加一个菜单项，链接到你的 Dify 嵌入页面（如 `https://your-dify-page.com`）。

---

### **2. 获取企业微信当前用户信息**

在企业微信中，可以通过 **JS-SDK** 或 **OAuth2.0** 获取当前用户信息（如 `user_id`）。

#### **方式一：使用 JS-SDK（推荐）**
```javascript
// 引入企业微信 JS-SDK
wx.config({
  beta: true,
  debug: false,
  appId: '你的企业微信应用ID',
  timestamp: '',
  nonceStr: '',
  signature: '',
  jsApiList: ['getCurExternalContact', 'getCurExternalChat']
});

wx.ready(function() {
  wx.getCurExternalContact({
    success: function(res) {
      let userId = res.userId; // 企业微信 user_id
      // 发送 userId 到你的后端，获取对应的 sales_id
    }
  });
});
```

#### **方式二：使用 OAuth2.0 静默授权**
- 在嵌入页面 URL 中使用企业微信 OAuth2.0 静默授权，获取 `code`。
- 后端使用 `code` 换取 `user_id`。

---

### **3. 后端映射 user_id → sales_id**

你需要一个后端服务（可集成到 CRM 系统或独立部署），用于：

1. 接收前端传来的 `user_id`。
2. 查询 CRM 数据库，获取对应的 `sales_id`。
3. 返回 `sales_id` 给前端。

---

### **4. 使用 sales_id 自动登录 Dify**

Dify 支持 **API Token** 或 **SSO 单点登录**（如果你已配置）。这里假设使用 **API Token** 方式：

#### **方案一：通过 Dify API 创建会话**
```javascript
// 使用 sales_id 调用 Dify API 创建或获取会话
fetch('https://dify.amer.cn/api/v1/chat/sessions', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_DIFY_API_KEY',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    user_id: sales_id, // 使用 sales_id 作为用户标识
    name: '企业微信用户'
  })
})
.then(response => response.json())
.then(data => {
  // 获取会话 ID，用于嵌入聊天框
});
```

#### **方案二：自定义嵌入页面并传递用户信息**
你可以在嵌入页面中预加载 Dify 聊天框，并在 URL 或 Cookie 中传入 `user_id`：

```html
<iframe 
  src="https://dify.amer.cn/chat/qQrRtKvsKyEmBoho?user_id=YOUR_SALES_ID"
  width="100%" 
  height="600px"
></iframe>
```

如果 Dify 支持 **JWT 或 Token 验证**，你还可以在 iframe 的 URL 中加入 token 参数。

---

### **5. 安全性建议**

- 建议在企业微信后端验证 `user_id`，不要在前端暴露 `sales_id`。
- 使用 HTTPS 确保传输安全。
- 建议为 Dify 设置 **IP 白名单**（仅允许企业微信和你的服务器访问）。

---

## 📌 **总结流程**

```
企业微信用户打开 CRM 菜单
    ↓
嵌入页面加载 → 企业微信 JS-SDK 获取 user_id
    ↓
发送 user_id 到后端 → 查询 CRM 映射 sales_id
    ↓
使用 sales_id 调用 Dify API 或嵌入聊天框
    ↓
Dify 聊天框自动以 sales_id 用户身份登录
```
