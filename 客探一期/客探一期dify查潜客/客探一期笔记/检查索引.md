• 你要的其实就是“轮询（polling）+ 循环（Loop）+ 等待（sleep）”这个模式：创建文档后先不要立刻去做知识检索/召回，而是先反复查询该文档的索引进度；如果还没完成就
  等一会儿再查，直到索引完成（或超时/失败）再继续往下走。

  下面给你两种实现方式，按你现在用的 API 返回字段来选。

  方案 A（推荐）：用 batch 轮询整批索引进度

  前提：你是用 Dify 的“创建文档”API（例如 create-by-text / create-by-file）创建的文档，这类接口响应里会返回一个 batch 字段，用来追踪索引进度。(docs.dify.ai

  对应的进度查询接口是：

  工作流节点搭法（Workflow / Chatflow 都适用）

  1. HTTP 请求节点：创建文档

  - 记住把响应里的 batch 保存下来（作为后面 Loop 的输入变量/循环变量）。响应里除了 document.id 也会有 batch。(docs.dify.ai (https://docs.dify.ai/api-
    reference/documents/create-a-document-from-a-file?utm_source=openai))

  2. （可选）HTTP 请求节点：添加/更新元数据

  - 这一步你已经做了。

  3. 循环（Loop）节点（用来“等到索引完成才放行”）
     Loop 节点支持：终止条件、最大循环次数、循环变量、以及在循环体内用 Exit Loop 节点直接退出。(docs.dify.ai (https://docs.dify.ai/en/use-dify/nodes/loop?
     utm_source=openai))
     建议配置：

  - Maximum Loop Count：比如 120（配合 2–3 秒间隔，大约等 4–6 分钟，避免无限等）
  - 循环变量（Loop Variables）：
      - attempt：0
      - wait_seconds：2（或 3）
      - batch：上一步创建文档返回的 batch
      - done：false（或者你用 Exit Loop，就不一定需要这个）

  4. Loop 循环体内部（每一轮做一次“查状态 → 判断 → 等待/退出”）：

  4.1) HTTP 请求节点：查询索引状态

  - Method: GET
  - URL: .../v1/datasets/{{dataset_id}}/documents/{{batch}}/indexing-status(docs.dify.ai (https://docs.dify.ai/api-reference/documents/get-document-
    embedding-status-progress))
  - Header: Authorization: Bearer {{API_KEY}}（Dify API Reference 都是这样鉴权）(docs.dify.ai (https://docs.dify.ai/api-reference/documents/get-document-
    embedding-status-progress))
  - 这个接口返回 data[]，每个元素有 indexing_status、completed_at、error、completed_segments、total_segments 等字段。(docs.dify.ai (https://docs.dify.ai/api-
    reference/documents/get-document-embedding-status-progress))

  4.2) 代码（Code）节点：把“是否完成/是否失败/进度”算出来
  Python 示例（输入变量映射：indexing_resp = 上一步 HTTP 的响应 body）：

  def main(indexing_resp: dict):
      items = (indexing_resp or {}).get("data") or []

      # 失败判定：只要有 error / stopped_at，就认为失败（你也可以按 indexing_status 判定）
      any_error = any((it.get("error") not in (None, "")) or (it.get("stopped_at") not in (None, 0)) for it in items)

      # 完成判定：优先看 indexing_status == completed；或者 completed_at 有值；或者进度满
      def is_done(it: dict) -> bool:
          status = it.get("indexing_status")
          if status == "completed":
              return True
          if it.get("completed_at") not in (None, 0):
              return True
          total = it.get("total_segments") or 0
          done = it.get("completed_segments") or 0
          return total > 0 and done >= total

      all_done = (len(items) > 0) and all(is_done(it) for it in items)

      # 可选：算个整体进度
      completed = sum((it.get("completed_segments") or 0) for it in items)
      total = sum((it.get("total_segments") or 0) for it in items)
      progress = (completed / total) if total else (1.0 if all_done else 0.0)

      return {
          "all_done": all_done,
          "any_error": any_error,
          "progress": progress,
      }

  （这里用到的字段都来自 indexing-status 接口响应结构。）(docs.dify.ai (https://docs.dify.ai/api-reference/documents/get-document-embedding-status-progress))

  4.3) If-Else 节点：分支控制

  - IF any_error == true：直接退出循环并输出错误（或走你的失败分支）
  - ELSE IF all_done == true：接一个 Exit Loop（或让 Loop 终止条件成立）(docs.dify.ai (https://docs.dify.ai/en/use-dify/nodes/loop?utm_source=openai))
  - ELSE：继续等待 → 下一轮

  4.4) 等待（sleep）
  如果你的节点列表里没有专门的“延迟/等待”节点，就用 Code 节点 time.sleep() 来等一会儿（这是最常见的做法）。

  import time

  def main(wait_seconds: int, attempt: int):
      time.sleep(wait_seconds)
      return {
          "attempt": attempt + 1,
          "wait_seconds": min(5, wait_seconds + 1),  # 简单退避：2s→3s→4s→5s封顶
      }

  然后回到循环体开头继续查。

  5. Loop 结束后再接你的“知识检索 / 后续处理”节点
     这样可以保证：只有当索引真的可用时才会往下执行。

  ## 循环体内部节点串联（从左到右）

  ### 1) HTTP 请求节点：GetDocDetail

  用途：每轮拉一次文档状态。

  - Method: GET
  - Path（示例写法）：

      -
        /v1/datasets/{{dataset_id}}/documents/{{document_id}}
  - Query Params（建议）：
      - metadata=without（轮询状态不需要元数据，减小响应体）(docs.dify.ai (https://docs.dify.ai/api-reference/documents/get-document-detail?
        utm_source=openai))
  - Headers：
      - Authorization: Bearer {{API_KEY}}(docs.dify.ai (https://docs.dify.ai/api-reference/documents/get-document-detail?utm_source=openai))
  - 重要：错误处理与重试（避免 loop 被网络抖动打断）
      - HTTP 节点支持 Retry on Failure（最多 10 次、间隔最大 5000ms）。(docs.dify.ai (https://docs.dify.ai/versions/3-0-x/en/user-guide/workflow/node/http-
        request?utm_source=openai))
      - 因为“Loop 内子节点一旦失败，Loop 会立刻停止并报错”，所以建议你给 HTTP 节点开启重试，并设置 Error Handling（默认值或 Fail Branch）来兜底。
        (docs.dify.ai (https://docs.dify.ai/en/use-dify/build/predefined-error-handling-logic?utm_source=openai))
      - 推荐组合：
          - Retry on Failure: 开启，max_retries=3，interval=1000ms
          - Error Handling: 选 Default Value，默认值给 {}（这样这轮拿不到状态也不会直接把 Loop 弄挂）

  > 说明：HTTP 节点输出里通常能拿到 body、status_code 等（文档里说明会返回响应体、状态码、头等）。(docs.dify.ai (https://docs.dify.ai/versions/3-0-x/en/user-
  > guide/workflow/node/http-request?utm_source=openai))

  ———

  ### 2) Code 节点：CheckIndexReady

  用途：把“文档详情响应”转换成几个明确的布尔值：ready / failed，以及给日志用的 status。

  Python 示例（输入变量按你实际选择映射；doc 指 HTTP 的 body）：

  def main(doc: dict):
      doc = doc or {}

      indexing_status = doc.get("indexing_status")
      display_status = doc.get("display_status")
      error = doc.get("error")
      enabled = doc.get("enabled", True)
      archived = doc.get("archived", False)

      # 索引可用：最稳的是 indexing_status == completed
      ready = (indexing_status == "completed") and (not error) and enabled and (not archived)

      # 明确失败：有 error 或者被禁用/归档（通常检索也用不上）
      failed = bool(error) or (indexing_status in ("error", "failed")) or (enabled is False) or (archived is True)

      status_for_log = indexing_status or display_status or "unknown"

      return {
          "ready": ready,
          "failed": failed,
          "status": status_for_log,
          "error": error,
          "enabled": enabled,
          "archived": archived,
          "indexing_status": indexing_status,
          "display_status": display_status,
      }

  字段依据来自“获取文档详情”的响应结构。(docs.dify.ai (https://docs.dify.ai/api-reference/documents/get-document-detail?utm_source=openai))
  另外，官方示例/文档里能看到 indexing_status 至少会出现 waiting 这类“未完成”状态，所以只要不是 completed，你就继续等即可。(docs.dify.ai (https://
  docs.dify.ai/versions/3-0-x/en/user-guide/knowledge-base/knowledge-and-documents-maintenance/maintain-dataset-via-api?utm_source=openai))

  ### 3) If-Else 节点：RouteByStatus

  三路分支：

  - IF：{{CheckIndexReady.failed}} == true
      - 直接走失败结束（End 节点/错误分支）
      - 建议把信息拼出来：status={{CheckIndexReady.status}}; error={{CheckIndexReady.error}}
  - ELIF：{{CheckIndexReady.ready}} == true
      - 先更新循环变量 ready=true（用于 loop 外做兜底判断），再进 Exit Loop

  - ELSE：（既没失败也没完成）

    utm_source=openai))
  - last_status: Overwrite 为 {{CheckIndexReady.status}}
  - last_error: Overwrite 为 {{CheckIndexReady.error}}（空就空）

  ### 5) Code 节点：Sleep

  用途：每轮没完成就睡一会儿，降低 API 压力。

  import time

  def main(seconds: int):
      seconds = int(seconds or 1)
      time.sleep(max(1, seconds))
      return {}

  输入 seconds 映射到 Loop Variables 的 sleep_seconds。

  当 ready==true 分支走到这里时立刻结束循环。Loop 支持 Exit Loop 作为立即终止条件之一。(docs.dify.ai (https://docs.dify.ai/en/use-dify/nodes/loop?

  ## Loop 节点外（很关键）：防止“到最大循环次数后误放行”

  Loop 到达 Maximum Loop Count 也会结束并继续往下走。(docs.dify.ai (https://docs.dify.ai/en/use-dify/nodes/loop?utm_source=openai))
  所以在 Loop 节点后面再接一个 If-Else：

  - IF {{ready}} == true：继续后续“知识检索/召回/问答”

      - timeout after {{attempt}} checks; last_status={{last_status}}; last_error={{last_error}}

  这样才能满足“索引不可用就等待，不会误继续”。

› Code 节点：CheckIndexReady的输出变量是什么，让我直接写入code节点


• 在 Dify 的 Code 节点里，“输出变量”就是你在 return { ... } 里返回的 字典 key；每个 key 都会变成该节点的一个输出字段，你在后续节点里用 {{节点名.key}} 引用 。

  下面这份 CheckIndexReady 代码，会输出 8 个变量：

  - ready（bool）：索引已可用
  - failed（bool）：索引失败/文档不可用（有 error / disabled / archived 等）
  - status（string）：用于日志显示的状态（优先 indexing_status，否则 display_status）
  - error（string/None）：接口返回的 error
  - enabled（bool）
  - archived（bool）
  - indexing_status（string/None）
  - display_status（string/None）

  你可以把下面整段直接粘进 Code 节点：

  def main(doc: dict):
      doc = doc or {}

      indexing_status = doc.get("indexing_status")
      display_status = doc.get("display_status")
      error = doc.get("error")
      enabled = doc.get("enabled", True)
      archived = doc.get("archived", False)

      # 索引可用：最稳的是 indexing_status == completed

      # 明确失败：有 error 或者被禁用/归档
      failed = bool(error) or (indexing_status in ("error", "failed")) or (enabled is False) or (archived is True)

      status_for_log = indexing_status or display_status or "unknown"

      return {
          "ready": ready,
          "failed": failed,
          "status": status_for_log,
          "error": error,
          "enabled": enabled,
          "archived": archived,
          "indexing_status": indexing_status,
          "display_status": display_status,
      }

  后续节点里这样用（假设 Code 节点名就叫 CheckIndexReady）：

  - 完成分支条件：{{CheckIndexReady.ready}} == true
  - 失败分支条件：{{CheckIndexReady.failed}} == true
  - 进度/日志：{{CheckIndexReady.status}}、{{CheckIndexReady.error}}

  如果你只想要最精简输出，也可以只 return {"ready": ..., "failed": ..., "status": ..., "error": ...}，其余 key 删掉即可。
