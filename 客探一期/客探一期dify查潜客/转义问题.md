• 你这个现象基本是转义/请求体格式导致的，不是提取逻辑本身问题。

  为什么只剩第一段

  - 你的 txt 里有大量双引号（"企业名称":"..." 这种）。
  - 在 HTTP 节点组装“创建文档 API”的 JSON body 时，如果变量没有被正确转义，后面的内容会被截断/解析失败，通常就只留下前面一小段。
  - 如果 HTTP 节点不是 application/json（比如表单模式），; 也可能被当分隔符，进一步导致只保留第一项。

  最稳的做法（推荐）
  把 txt 改成“每行一个字段”，不要再用 " 和 ;。知识库检索效果也更好。

  把代码最后输出改成这样（仍然只输出 txt）：

  lines = [f"{k}：{result.get(k, '-')}" for k in target_keys]
  txt = "\n".join(lines)
  return {"txt": txt}

  HTTP 节点要这样配

  - Header：Content-Type: application/json
  - Body（Raw JSON）示例：

  {
    "name": "企业档案",
    "text": "{{#上一个代码节点.txt#}}"
  }
