关键点先说清楚：`Code` 节点负责“整理数据并输出数组”，不要在 `Code` 节点里直接异步建文档。按照 Dify 当前文档，`Code` 节点运行在沙箱里，不能发网络请求；真正的创建/更新文档，应该放到 `Iteration` + `HTTP Request` 节点里做。`Iteration` 支持并行，最多 10 个并发，所以你要的“存进数组后异步创建”是可行的。  
另外，我按你当前的 `example.json` 实际跑了一次，不止会生成 `员工ID + 2026-W10` 和 `员工ID + 2026-W11`，还会多一个 `员工ID + 2026-W09`，因为文件里还有 `2026-02-26`、`2026-02-27`、`2026-02-28` 的记录。

**Code 节点示例**
不要只存文档名字符串，最好直接输出“文档对象数组”，每个元素都带上 `document_name` 和 `content`。

```python
import json
from datetime import datetime, timedelta

def main(api_response, employee_id: str) -> dict:
    if isinstance(api_response, str):
        api_response = json.loads(api_response)

    weekly_docs = {}

    for customer_id, customer in api_response.get("data", {}).items():
        customer_name = (customer.get("customer_name") or "").strip()

        for report in customer.get("customer_CallReport", []):
            visit_dt = datetime.strptime(report["time"], "%Y-%m-%d %H:%M:%S")
            visit_date = visit_dt.date()

            iso_year, iso_week, _ = visit_date.isocalendar()
            week_label = f"{iso_year}-W{iso_week:02d}"
            document_name = f"{employee_id} + {week_label}"

            week_start = visit_date - timedelta(days=visit_date.weekday())
            week_end = week_start + timedelta(days=6)

            doc = weekly_docs.setdefault(document_name, {
                "document_name": document_name,
                "week_label": week_label,
                "week_start": week_start.isoformat(),
                "week_end": week_end.isoformat(),
                "days": {}
            })

            doc["days"].setdefault(visit_date.isoformat(), []).append({
                "customer_id": customer_id,
                "customer_name": customer_name,
                "visit_way": (report.get("visit_way") or "").strip(),
                "summary": (report.get("summary") or "").strip(),
                "time": report["time"]
            })

    documents = []

    for document_name in sorted(weekly_docs):
        doc = weekly_docs[document_name]
        content_parts = []
        day_list = []

        for day in sorted(doc["days"]):
            records = sorted(doc["days"][day], key=lambda x: x["time"])
            day_list.append({
                "date": day,
                "records": records
            })

            lines = [f"：{day}"]
            for item in records:
                lines.append(f"客户名称：{item['customer_name']}")
                lines.append(f"拜访方式：{item['visit_way']}")
                lines.append(f"拜访总结：{item['summary']}")

            content_parts.append("\n".join(lines))

        content = "\n\n".join(content_parts)

        documents.append({
            "document_name": document_name,
            "week_label": doc["week_label"],
            "week_start": doc["week_start"],
            "week_end": doc["week_end"],
            "days": day_list,
            "content": content,
            "create_payload": {
                "name": document_name,
                "text": content,
                "indexing_technique": "high_quality",
                "process_rule": {"mode": "automatic"}
            },
            "update_payload": {
                "name": document_name,
                "text": content,
                "process_rule": {"mode": "automatic"}
            }
        })

    return {
        "document_names": [d["document_name"] for d in documents],
        "documents": documents,
        "total_documents": len(documents)
    }
```

这段跑完后，`documents` 大概就是这种结构：

```json
[
  {
    "document_name": "员工ID + 2026-W10",
    "content": "：2026-03-02\n客户名称：...\n拜访方式：...\n拜访总结：...\n\n：2026-03-06\n客户名称：...\n..."
  },
  {
    "document_name": "员工ID + 2026-W11",
    "content": "：2026-03-09\n客户名称：...\n拜访方式：...\n拜访总结：..."
  }
]
```

**Dify 工作流建议**
1. `Code` 节点输出 `documents`
2. `Iteration` 节点输入 `documents`
3. 打开 `Parallel Mode`
4. 迭代内部先用 `HTTP Request` 调 `GET /datasets/{dataset_id}/documents?keyword={{items.document_name}}`
5. `IF/ELSE` 判断是否存在同名文档
6. 存在就调 `POST /datasets/{dataset_id}/documents/{document_id}/update-by-text`
7. 不存在就调 `POST /datasets/{dataset_id}/document/create-by-text`

建议判断“是否存在”时，不要只看 `keyword` 搜到了结果，还要再比一次 `name == items.document_name`，因为 `keyword` 是模糊匹配。

创建文档的请求体：

```json
{
  "name": "{{items.document_name}}",
  "text": "{{items.content}}",
  "indexing_technique": "high_quality",
  "process_rule": {
    "mode": "automatic"
  }
}
```

更新文档的请求体：

```json
{
  "name": "{{items.document_name}}",
  "text": "{{items.content}}",
  "process_rule": {
    "mode": "automatic"
  }
}
```

还有一个很重要的点：`update-by-text` 本质上是“整篇更新”，不是“自动追加一段”。所以如果你的接口每次返回的是“本周全量数据”，那直接覆盖更新最简单；如果每次只返回“新增拜访”，那你更新前还要先把旧文档内容取回来再合并。
