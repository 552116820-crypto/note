● 问题出在 URL 中的 + 号。在 URL 查询参数里，+ 会被解析为空格，所以实际搜索的关键词变成了 001708   2026-W09，而不是
  001708 + 2026-W09。

  另外 Dify API 的 keyword 参数是单关键词模糊搜索，不支持多条件。

  推荐方案

  把 URL 中的 keyword 只用一个条件（比如 employee_code），粗筛出候选文档，然后在后面的代码节点里做精确匹配：

  HTTP 请求节点 URL 改为：
  http://dify.amer.cn/v1/datasets/{{#env.dataset_id#}}/documents?keyword={{#1774488119739.employee_code#}}

  只用 employee_code 搜索（少量结果），然后在 "代码执行 6" 节点中精确匹配完整的 document_name：

  import json

  def main(body, document_name):
      docs = json.loads(body)['data']

      # 精确匹配完整文档名（如 "001708 + 2026-W09"）
      target_names = set(document_name)

      found = False
      for doc in docs:
          if doc['name'] in target_names:
              found = True
              break

      return {
          "exists": found
      }

  原因总结

  ┌─────────────────────────────────┬──────────────────────────────────────────────────────────────────────┐
  │           URL 中写的            │                             实际被解析为                             │
  ├─────────────────────────────────┼──────────────────────────────────────────────────────────────────────┤
  │ keyword=001708 + 2026-W09       │ keyword=001708   2026-W09（+ 变空格）                                │
  ├─────────────────────────────────┼──────────────────────────────────────────────────────────────────────┤
  │ keyword=001708%20%2B%202026-W09 │ keyword=001708 + 2026-W09（正确，但 Dify 模板变量不会自动 URL 编码） │
  └─────────────────────────────────┴──────────────────────────────────────────────────────────────────────┘

  所以最稳妥的做法就是：keyword 只传一个简单值做粗筛，代码节点里做精确的名称匹配。
