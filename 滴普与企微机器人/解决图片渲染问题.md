**为什么 LLM 会把 "link" 当字面值输出**

这两行的本意是教模型「不管 src 里实际是什么 URL，都按这种格式渲染」，把 `link` 当成元变量。但 LLM 经常把 few-shot 示例里的字符串**当字面值复用**，尤其是：
- 上下文里没有真正的图片链接
- 但 prompt 又强制要求"不能省略 link"（第 4 条规则）
- 模型为了同时满足两个指令，就把示例里的 `link` 原样输出了

加上 `link` 这个词在英语里既是普通名词又是变量名，更容易被复制。这就是为什么气泡里出现了 `<img src="link"alt="" />`，最后掉到机器人 `图片 2 加载失败：link` 的兜底里。

**协议限制**：`aibot_respond_msg` 每条消息只能带一个 `msgtype`（stream/text/image/file/markdown/news 等），不能像 IM 富文本那样在同一气泡里 text + image 混排。我们目前的做法就是这个 API 唯一的"图文同步"模式——文字气泡里用 `[图片N]` 占位说明这里有图，紧跟一条独立 image 气泡承载真图。中间的 `以下为新消息` 是企微 UI 自己的「滚动到新消息」分隔条，不是我们插的。

1. **去掉 `[图片N]` 占位**：如果图片基本都在答案末尾（你这个截图的情况），占位词显得多余，可以让 `extract_image_refs` 替换成空串。多图穿插场景下占位词能帮用户对齐位置，单图场景就纯噪声。

3. **news 卡片**：可以做一张「标题 + 描述 + 缩略图 + 跳转链接」的卡片，但只能展示一张缩略图，长表格这种场景不合适。
