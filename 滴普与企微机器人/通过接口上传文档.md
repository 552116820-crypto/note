• 对于 Dify 这一类工作流，结论是：

  文档类文件不能直接丢给大模型节点指望它自动读内容。
  上传文件变量本身通常只是文件对象/引用，不等于文件正文。Dify 官方文档也明确说明：文件上传需要通过 Document Extractor /
  文档提取器 或其他工具，把文件内容转换成 LLM 能理解的文本，再传给大模型节点。文档提取器输出的 text 才是给 LLM 读的内
  容。

  适用方式一般是：

  1. 开始节点接收文件变量。
  2. 如果是数组文件，先用 List Operator/迭代拆文件。
  3. 文档文件走 文档提取器，输出 text。
  4. 在大模型节点 prompt 里引用提取结果，比如 {{doc_extractor.text}}。
  5. 图片类文件可以走视觉模型直接理解，但 PDF/Word/TXT 这类文档仍建议先解析成文本。

  所以你问的“不在此之前做文档解析，模型能读到文件内容吗”：
  通常读不到，最多只能拿到文件引用、文件名、类型、URL/id 之类信息，读不到 PDF 正文。

  这也是我之前在代码里做“上传文件 + 同时把本地抽取的文本追加进 query”的原因：防止滴普/Dify 工作流没有正确消费文件变量
  时，大模型节点只看到文件对象，看不到文档正文。
