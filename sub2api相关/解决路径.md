❯ 总结一下，你是怎么解决这个问题的

● 问题排查与解决总结

  1. 登录密码丢失

  - .env 中 ADMIN_PASSWORD 为空，首次启动时自动生成的密码日志已丢失
  - 密码以 bcrypt 哈希存储在数据库，无法逆向
  - 解决：用 Python 生成新的 bcrypt 哈希，直接 UPDATE users SET password_hash = '...' 更新数据库

  2. INSUFFICIENT_BALANCE 余额不足

  - 管理员账户余额为 0
  - 解决：UPDATE users SET balance = 999999 WHERE id = 1

  3. 上游账户未配置

  - sub2api 是把 ChatGPT/Claude 订阅转成 API 的工具，需要绑定上游订阅账号
  - 解决：你在管理后台通过 OAuth 添加了 ChatGPT Plus 账号

  4. 账户未分配到分组

  - API key 属于一个分组，上游账户必须也在同一分组中才能被调度
  - 直接改数据库不会触发调度缓存更新
  - 解决：通过管理 API (PUT /api/v1/admin/accounts/4) 正确分配分组

  5. 分组模型范围限制

  - 分组的 supported_model_scopes 只允许 ["claude", "gemini_text", "gemini_image"]，排除了 OpenAI 模型
  - 解决：清空为 []，允许所有模型

  6. 容器无法访问外网

  - 上游请求需要代理才能访问 chatgpt.com
  - 全局 HTTP_PROXY 环境变量对 sub2api 的上游请求无效，需要在应用内配置
  - 解决：通过管理 API 创建代理 (host.docker.internal:7890) 并绑定到账户

  7. 模型名和响应格式

  - openai/gpt-oss-120b:free 不是 ChatGPT 支持的模型，改用 gpt-5.4
  - 非流式模式返回内容为空，需要用 stream=True

  核心教训

  ▎ 直接改数据库容易导致缓存不同步，应优先使用管理后台或管理 API 操作。
