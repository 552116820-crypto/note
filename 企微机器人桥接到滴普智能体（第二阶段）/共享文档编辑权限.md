### Turn 1  *(2026-06-10 03:43)*

**User:**

现在能通过企微机器人直接将信息录入进企微的共享文档中吗

**Assistant:**

查清楚了，结论如下。

## 能，但有一个关键限制：只能写机器人"自己创建"的文档

企业微信近期给 **API 模式智能机器人**（正是本项目用的这种，`WECOM_BOT_ID` 长连接那个体系）开放了文档能力：成员在「工作台 → 智能机器人 → 编辑 → 可使用权限」里给机器人授权后，机器人可以：

- `create_doc` — 新建文档或智能表格（返回 docid 和链接）
- `edit_doc_content` — 写入文档内容，**支持 Markdown**
- `smartsheet_add_records` / `add_sheet` / `add_fields` 等 — 往智能表格里加子表、字段、记录

所以"机器人把信息录入企微文档"这件事现在是官方支持的。但有三个限制要先知道：

1. **只能编辑机器人自己创建的文档**。官方原文："机器人仅可编辑自己创建的文档"。也就是说，如果你想录入的是某个同事手工建好的现有共享文档，**做不到**。自建应用的传统文档 API 也是同样限制（"应用仅可读取和编辑自己创建的文档，无法编辑成员创建的文档"），这条路绕不过去。
2. **目前只支持 MCP（streamableHTTP）方式调用**，不是普通 HTTP API。授权后在机器人编辑页能拿到一个 streamableHTTP URL / JSON Config。
3. **授权有效期只有 7 天**，到期要人工重新授权——对无人值守的生产服务不太友好，这是目前最大的工程痛点。

另外 2026 年 4 月的更新里，机器人还获得了"**读取**成员文档"的能力：成员授权后，机器人可以根据成员发来的文档链接读取内容——但注意这只是读，写入仍然限于自己创建的文档。

## 推荐的落地模式

既然不能写入已有文档，实际可行的做法是反过来：**让机器人先 `create_doc` 建一个文档或智能表格，把链接分享到群里，以这个文档作为大家的"共享文档"**，之后机器人就能持续往里录入了。流水账类信息（会议记录、问题登记、数据上报）用智能表格 + `smartsheet_add_records` 比纯文档更合适。

对接到本项目有两个位置可选：

- 在桥接服务里自己实现一个 MCP client，收到特定指令（比如群里的命令前缀路由）时调 `edit_doc_content` / `smartsheet_add_records`；
- 或者如果滴普 Agent 平台支持挂载 MCP 工具，直接把企微给的 streamableHTTP URL 配到 Agent 上，由 Agent 自主决定何时写文档——这样桥接服务不用改。

## 对"持续录入"场景的两个设计要点

1. **录入类信息用智能表格，不要用文档**。`edit_doc_content` 是**整篇覆写**语义，而且这 16 个工具里**没有读取文档内容的接口**——Agent 没法"读旧内容再追加"，多轮录入文档必丢数据。智能表格的 `smartsheet_add_records` 是追加一行，天然适合录入（会议记录、问题登记、数据上报都是这个模式）。
2. **把目标表格的 docid/链接固定下来**。每次让 Agent 现找文档不现实，建议建好表后把 docid 或 URL 写进滴普 Agent 的系统提示词里（"录入问题时写入这个表格：URL=..."），这样用户只需要说"记一条：xxx"。

另外两个运维提醒：7 天授权过期是常态化问题（你今天就撞上了），上线后要安排人定期续期，并让 Agent 把 850003 的报错原样透传到群里好让人知道该续了；还有这个 apikey 是写你企业文档的凭证，已经出现在聊天记录里了，如果这个会话之外还贴过其他地方，建议续期时直接换新的。
