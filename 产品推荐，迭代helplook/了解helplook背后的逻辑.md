## 四、几个值得单独点出的判断

1. **deepseek 只管"生成"，完全不碰检索。** 它的角色就是阅读理解 + 写作。

2. **embedding 一定不是 deepseek 干的**，两条理由：
   - (a) 官方让你在 GPT/Claude/豆包/文心/deepseek 之间换生成模型，但**向量索引是发布时就建好的、不会因为换模型重建** → 说明 embedding 跟生成模型是**解耦**的；
   - (b) DeepSeek 开放平台到目前**没有公开的文本 embedding 接口**（只有 deepseek-chat / deepseek-reasoner）。
   
   所以向量层用的是**另一个 embedding 模型**（HelpLook 自己定的，中文内容大概率是 BGE/M3E 这类或某云厂商的 embedding），和你看到的 "deepseek-chat" 是两码事。

3. **deepseek-chat = V3 这条非推理线**（区别于 deepseek-reasoner/R1）。它"没有就说没有"的克制**不是思维链推出来的，是 prompt 逼出来的**；表格也只是格式化能力。HelpLook 选它就是图便宜、快，RAG 总结这活够用了。

4. **这套东西的"智能"上限由检索质量决定，不由 deepseek 决定。** 答案要是混进不相关的油、或漏掉某个该推的型号，九成是切片/召回的问题，不是模型笨。优化方向是知识库切片和检索，不是换更强的 LLM。

## 新刷新出来的三点

1. **TopK 不是定死的：12 vs 9。** 更像"取 top-N 候选 + 相似度阈值过滤"——模糊 query 能过阈的切片少，所以剩得少。不是每次硬返回固定条数。

2. **chunk 级检索石锤。** "液压油产品特点和选择推荐"一篇就贡献了 **4 个片段**（片段1~4），一篇文档的 4 块各自独立命中。比第一问的 2 片段证据更硬。

3. **召回的文档体裁跟着 query 漂。** 第一问几乎全是规格表，第二问几乎全是叙述/营销文。检索器不挑体裁，只看相关度，所以"问得像什么，就捞回什么调性的料"。

**A. 护栏当着你的面把竞品过滤掉了。** 召回里有 `TDS-卓力L-HV`、`TDS-卓力L-HS`、还有一篇"竞品分析"——但答案只推安美 XY-HV32，还明说"此为安美自有品牌，首选推荐"。**竞品数据进了上下文，护栏把推荐口径拐回自家。**（卓力是友商还是安美子品牌？你应该一眼知道——若是友商，这就是品牌护栏在硬过滤，护栏强度比我原来估的还高。）

你问的是"卖得好"，它推了个**低温油**（倾点 -52℃），而召回里也清一色低温（卓力 L-HV 低温 / L-HS 超低温）。说明安美这个库的"液压油"内容本身就**偏重低温这条产品线**。所以它推的根本不是"客观最畅销"，是"**这个库里被讲得最多、被框成主推的那个**"。
