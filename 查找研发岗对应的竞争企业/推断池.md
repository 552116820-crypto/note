    P04 推断池 = 没有公开展商目录可抓时，按"行业头部画像"主动推断可能出现的厂家清单，作为兜底数据源。

    为什么需要推断池

    P04 特种润滑脂这次想跑 8 个展会，但其中 LB02 义乌 / LB04 武汉 / LB05 郑州 / LB06 ILExpo 4 个展会的展商目录在公网上完全不可达：
    - 官网 SSL 自签证书 / 证书过期
    - 第三方汇总站（jufair / shifair / fair51）只有展会简介，不公开展商名录
    - 付费购买 ¥75（jufair LB08 那种）你没买
    - 找展商目录搜了 N 个源都返回"未列具体公司名"

    不可达就没原始证据可抓。推断池就是这种情况的兜底方案。

    推断池怎么构造

    不去抓不存在的展商目录，而是按 P04 岗位的品类画像 推断「这种岗位的展会大概率会出现的厂家」：

    - 全球润滑脂龙头：Klüber / Total / PETRONAS / Idemitsu / ENEOS / Repsol 等
    - 中国润滑油脂头部：龙蟠 / 统一 / 高桥石化 等
    - 行业 top 报告里的品牌：chinabgao 2025 / cnpp 2026 十大榜（协同油脂 / 海华 / 雪佛龙 / Valvoline / MOTUL / 康普顿 等）

    关键标记 is_inferred=true

    推断池产出的每家公司在 exhibit_sources 里必须标 is_inferred=true，并在 evidence_quote 写明「行业推断池，非官方展商目录」。HR 看 xlsx 时一眼能区分：
    - 直接证据：来自 LB03 sina 报道 / LB01 vision 识图 / LB08 jufair 示例 — is_inferred=false
    - 推断：来自 batch3 + batch4 — is_inferred=true

    局限

    推断池不替代实地展商目录。这次 P04 跑出来的 verified.jsonl 里，推断池部分严格说是「行业内大概率参展的厂家」而非「2024 届实际在 LB02/04 这些展会出现的厂家」。等用户下次拿到具体展会会刊或现场拍照后，可以走
    apply_decisions 把推断转为坐实证据。

    简言之：推断池 = 用"行业头部清单"代替"没法访问的展商目录"，标 is_inferred=true 让 HR 知道这是推断不是抓取。
