**User:**

**Assistant:**

**SPA = Single Page Application（单页应用）**

一种现代网站架构。简单说：

| | 传统网站 | SPA |
|---|---|---|
| 服务器返回的 HTML | 含全部内容（公司名、列表、表格都在里面） | 几乎是空架子，只有 `<div id="app"></div>` |
| 真正的内容 | 直接在 HTML 里 | **由浏览器跑 JS 后从后端 API 拉数据，再动态填进 DOM** |
| 翻页 | 整页刷新 | URL 变了但页面不重载，前端调 API 换数据 |
| view-source 看到的 | 完整内容 | 空架子 + 一堆 JS 文件引用 |

**productronica 就是典型 SPA**：

```
你 WebFetch 拿到的 HTML（约 5KB）:
<html>
  <div id="app">Loading...</div>
  <script src="main.xxx.js"></script>
</html>
                            ↓ 浏览器跑 JS
                            ↓ JS 调用 /api/exhibitors 拿数据
                            ↓ JS 把数据填进 DOM
真实渲染后的 DOM（约 200KB）:
<div class="content-item">亚德客 / AirTAC ...</div>
<div class="content-item">ABB ...</div>
... 15 家 ...
```

WebFetch 只下载第一步那 5KB，永远拿不到 15 家公司名。所以必须用**真浏览器**（playwright 起的 headless chromium）跑完 JS 之后再读 DOM。

**怎么一眼看出是 SPA**：
- 浏览器开站后右键"查看页面源代码"，源代码里搜不到页面上看到的公司名
- 或者你能在站点里来回点击导航，但浏览器地址栏一直没整页刷新（只是 URL 变）

国内常见 SPA：productronica / NEPCON / HKPCA 官网、京东、抖音 web 端、所有 Vue/React/Angular 开发的现代站。
