# IsolationForest介绍

## 第1轮

**用户：** IsolationForest介绍一下

**助手：**

`Isolation Forest` 是一种常用的**无监督异常检测算法**，核心思想很直观：

**异常点更容易被“孤立”出来。**

正常样本通常分布在大块区域里，要切很多次才能单独分出来；异常样本往往离群，只要随机切几刀就能被单独隔离出来。

**它怎么做**
1. 随机抽一部分样本。
2. 随机选一个特征。
3. 在该特征的取值范围内随机选一个切分点。
4. 不断递归切分，生成很多棵随机树。
5. 看一个样本在这些树里平均经过多少层才被隔离。

如果一个点的平均路径很短，说明它很容易被切出来，越可能是异常点。

**为什么叫 Forest**
因为它不是只建一棵树，而是建很多棵“孤立树” (`Isolation Tree`)，最后综合多棵树的结果，提高稳定性。

**优点**
- 不需要标注数据
- 对大规模数据比较友好
- 训练和预测通常都比较快
- 对高维数据也常能工作
- 比基于距离的方法更不容易受特征尺度影响

**缺点**
- 本质是随机切分，结果会有一定波动
- 如果异常点并不稀疏、而是形成一小团，效果可能没那么理想
- 对类别特征、强业务规则数据，通常需要额外预处理

**常见参数**
- `n_estimators`：树的数量
- `max_samples`：每棵树用多少样本
- `contamination`：预估异常比例，用来决定阈值
- `random_state`：固定随机种子，便于复现

**适用场景**
- 欺诈检测
- 网络入侵检测
- 设备故障预警
- 日志异常发现
- 数据清洗中的离群点识别

## 第2轮

**用户：** 图示版解释

**助手：**

把 `IsolationForest` 想成：在散点图上**随机切很多刀**，看哪个点最容易被单独切出来。

**1. 数据长什么样**

```text
y
^
|                x
|
|      o o o o o
|    o o o o o o
|      o o o o
|
+--------------------------> x
```

- `o`：大多数正常点，挤在一起
- `x`：一个离群点，离大家很远

这里直觉上就能看出来：`x` 更容易被单独分出来。

**2. 随机切第一刀**

比如随机选了 `x轴`，在某个位置竖着切：

```text
y
^
|                x
|              |
|      o o o o | o
|    o o o o o | o
|      o o o o |
|
+--------------------------> x
```

这一刀已经把右边那小块区域切出来了，`x` 更接近被单独隔离。

**3. 再随机切一刀**

再来一刀，可能选 `y轴` 横着切：

```text
y
^
|-------------- x
|
|      o o o o o
|    o o o o o o
|      o o o o
|
+--------------------------> x
```

这时 `x` 很快就被单独框出来了。

**4. 正常点为什么不容易被隔离**

看中间那团正常点：

```text
y
^
|
|      o o o o o
|    o o o o o o
|      o o o o
|
+--------------------------> x
```

因为它们挤在一起，一刀两刀通常只能把“大块区域”切开，切不出单个点。  
所以要切很多次，某个正常点才会被单独分出来。

**5. 树结构里怎么理解**

每切一次，相当于树多一层：

```text
根节点
├─ 左边区域：很多正常点
└─ 右边区域：只剩少量点
    └─ 再切一次
        └─ x 被单独隔离
```

- 异常点：树路径短，几层就隔离了
- 正常点：树路径长，要很多层

**一句话记忆**

`IsolationForest` 的核心就是：

- **越异常的点，越容易被随机切分单独隔离**
- **平均隔离路径越短，异常分数越高**
