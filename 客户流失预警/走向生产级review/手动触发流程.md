**手动触发 Celery 任务**

先确保 Redis 和 Celery worker 在跑。手动触发不一定需要 beat，只需要 worker：

```bash
celery -A app.celery_app:celery_app worker --loglevel=info
```

另开一个终端执行：

```bash
celery -A app.celery_app:celery_app call app.churn.tasks.run_detection --args='["frequency"]'
```

其他类型：

```bash
celery -A app.celery_app:celery_app call app.churn.tasks.run_detection --args='["product"]'
celery -A app.celery_app:celery_app call app.churn.tasks.run_detection --args='["volume"]'
```

如果要指定业务日期，传第二个参数：

```bash
celery -A app.celery_app:celery_app call app.churn.tasks.run_detection --args='["frequency","2026-04-18"]'
celery -A app.celery_app:celery_app call app.churn.tasks.run_detection --args='["product","2026-04-18"]'
celery -A app.celery_app:celery_app call app.churn.tasks.run_detection --args='["volume","2026-03-31"]'
```

注意：`volume` 建议传月末日期；不传时项目会自动用上一个完整自然月的月末日期。

**Docker Compose 环境**

如果是 compose 部署：

```bash
docker compose exec celery celery -A app.celery_app:celery_app call app.churn.tasks.run_detection --args='["frequency"]'
```

同理可换成 `product` 或 `volume`。

**查看结果**

`celery call` 会返回 task id。配置了 `CELERY_RESULT_BACKEND` 时可以查：

```bash
celery -A app.celery_app:celery_app result <task-id>
```

这个任务执行完成后也会保存 detection run，可以通过 API 查最近运行记录：

```bash
curl -H "X-API-Key: $API_KEY" "http://localhost:8000/api/detection-runs?limit=5"
```

如果本地没设置 `API_KEY`，可以不带 header。

**另一种手动方式：走 HTTP 批处理接口**

项目也有手动分步接口，但它不是“一键跑完整 Celery 任务”。流程是：

1. `POST /api/warning-jobs/start`
2. `GET /api/warning-jobs/{job_id}/batches`
3. 对每个 batch 调 `POST /api/warning-jobs/{job_id}/detect-batch`

这些接口在 [app/churn/routes.py](/mnt/e/churn-warning-engine/app/churn/routes.py):122 和 [app/churn/routes.py](/mnt/e/churn-warning-engine/app/churn/routes.py):152。
