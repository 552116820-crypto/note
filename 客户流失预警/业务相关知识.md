**Assistant:**

这个 8.67 天是通过 `predict_cycle` 算法算出来的，流程如下：

### 算法步骤

以这个客户为例（3 笔订单）：

**1. 计算下单间隔**
假设 3 笔订单日期排序后，相邻两笔之间算出 2 个间隔天数，比如 `[10天, 8天]`

**2. IQR 去异常值**
数据不足 4 条时跳过这步

**3. EWMA 指数加权平均**
越近的间隔权重越大（权重为 `2^i`）：
```
权重: [2^0, 2^1] = [1, 2]
EWMA = (10×1 + 8×2) / (1+2) = 26/3 = 8.67
```

所以 **8.67 天就是加权平均后的预测下单周期**，近期的间隔占更大比重。

**4. 判断是否预警**
- 预测周期：8.67 天
- 实际已 80 天没下单
- 80 > 8.67 × 1.5（频率倍数阈值）→ 触发预警
- 偏离比：80 / 8.67 = 9.23 倍 → severity = high

### 为什么置信度只有 0.35？

因为只有 3 笔订单（样本少），置信度算法认为数据不够充分，预测可靠性偏低。订单越多、间隔越稳定，置信度越高。

核心代码在 `app/churn/engine.py` 的 `predict_cycle` 和 `_ewma` 函数。
