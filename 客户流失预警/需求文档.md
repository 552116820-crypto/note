## 一、需求背景

在CRM的运营中，客户采购行为的稳定性直接影响公司营收。当前对于老客户下单行为的监测仍依赖人工或被动数据统计，缺乏主动、智能的异常识别机制，难以应对客户被竞品争夺的动态市场环境。

## 二、业务痛点

● 客户被竞品 “切走” 时，销售团队后知后觉，错失挽回时机，导致客户流失率居高不下。

● 对客户下单频率、量级、成品的变化缺少及时感知，无法提前预判风险，被动陷入 “客户流失后再补救” 的低效循环。

## 三、需求价值

### （一）业务价值

1.  提升客户留存率：通过提前预警客户流失风险，销售团队可针对性介入，有效挽回高风险客户，降低客户流失率。

2.  优化资源分配：将销售资源精准投入到高风险客户群体，避免精力分散，提升资源利用效率。

3.  增强市场竞争力：主动掌握客户采购动态，在竞品争夺中抢占先机，巩固公司在供应链中的地位。

### （二）长期价值

● 延长客户生命周期价值，提升客户满意度。

● 推动 CRM 从 “数据记录工具” 向 “智能决策工具” 升级，赋能数字化运营能力。

## 四、具体需求内容

基于 “异常识别 - 主动预警 - 及时干预” 的逻辑，针对以下三类场景实现 AI 主动监测与预警。

移动端有高风险客户预警大屏，方便业务领导查看并分析。

模型的指标支持自定义设置，AI给出系统参考值。

### （一）频率异常预警

● 需求痛点：客户是每月下单的，X天仍未下单，触发预警。

● 技术方案：AI分析客户下单历史规律，预测下单周期；当超出预测的下单周期2倍时间的时候触发预警，支持修改触发预警的时间。

● 需求说明：企业微信通知对应销售，提示 “【客户流失预警】客户 XXX 已超 X 天未下单，请及时跟进”；这里的时间是从客户上次下单时间开始算起。

● 销售都收到提醒，且都可以点进小程序消息去修改预警配置。

通知时间：每周一早上8点提醒

● 只要填上倍数，自动计算预警时间。比如AI预测时间：30天，需要在第45天发消息提醒则填1.5倍。

### （二）量级异常预警

● 需求痛点：对比客户上月下单量环比下降 50% 及以上，触发预警。

● 技术方案：当本月下单数量环比下降50% 及以下或是超出50% 及以上触发预警，支持修改下降或是超出比例；每个月月初第一天早上8点提醒。

● 需求说明：企业微信通知对应销售，提示 “【客户流失预警】客户本月下单数量下降60%，请及时跟进”。

● 销售都收到提醒，且都可以点进小程序消息去修改预警配置。

通知时间：每周一早上8点提醒

● 超出/下降比例：灰色数字代表是最近一次的数据，可修改

### （三）产品异常预警

● 需求痛点：识别客户停止采购的 “关键产品”（定义为占客户当年度总采购额 30%及以上），一旦停止采购即触发预警。

● 技术方案：AI分析客户关键产品下单历史规律，预测下单周期；当超出预测的下单周期2倍时间的时候触发预警，关键产品的占比和预警时间支持自定义修改。

● 需求说明：企业微信通知对应销售，提示 “【客户流失预警】客户 XXX 已超 X 天未下单某产品，请及时跟进”；这里的时间是指关键产品的时间，从客户上次下单关键时间开始算起。

● 销售都收到提醒，且都可以点进小程序消息去修改预警配置。

通知时间：每周一早上8点提醒

● 只要填上倍数，自动计算预警时间。比如AI预测时间：30天，需要在第45天发消息提醒则填1.5倍。

● 比如关键产品占比30%，那该客户采购的产品中只要占他过去一年采购金额30%及以上的产品就要预警。这里的预警是关键产品，上面那条预警是所有的产品，预警的时间可以设置不同。

● 某某产品占比：灰色数字代表是最近一次的数据，可修改

### （四）客户流失预警大屏

● 需求痛点：所有的预警信息没有汇总及统一查看的地方。

● 技术方案：安美伙伴移动端和BI系统增加客户流失预警大屏，方便业务领导查看及会议沟通。

● 需求说明：在安美伙伴移动端反馈那里增加一个客户流失预警大屏，页面如下图。一级大屏展示目前的消息（5条），排序按照时间越新排越前，二级大屏是展示单个类目的详细记录，排序规则一致。

大屏目前需要支持手机端和电脑端，手机端放在安美伙伴，电脑端嵌入到BI系统

## 五、实施计划

|   |   |   |
|---|---|---|
|阶段|时间周期|核心动作|
|需求评审|1 周|联合销售、算法、数据团队评审需求细节，确定各场景阈值（如 “X 天”“50%” 的具体数值）|
|技术方案设计|2 周|算法团队对接 CRM 订单、成品数据，设计异常识别模型；产品团队输出原型与交互逻辑|
|开发与测试|4周|完成功能开发，开展数据模拟测试与真实场景灰度测试，验证预警准确率与及时性|
|上线与迭代|持续|全量上线后收集用户反馈，优化预警规则与交互体验，迭代功能|

## 六、业务价值

● 效率提升：客户流失预警响应时间从 “事后被动发现” 缩短至 “提前主动预警”。

● 留存增长：预计客户留存率提升，因客户流失导致的年度营收损失减少。

● 能力沉淀：建立可复用的客户行为异常识别模型，为后续供应链风险、库存风险等场景的智能预警奠定基础。
