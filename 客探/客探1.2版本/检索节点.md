• 最稳的做法是把“周报取原文”从 知识检索 改成“两次 API + 两个代码节点”：

  1. 先算出标准化的 week_label 和 document_name
  2. 用文档列表 API 按 document_name 找到唯一 document_id
  3. 再用 chunks API 把这个文档的所有段落取出来
  4. 拼成 doc_text 后喂给 LLM

  你这套周报很适合这样做，因为文档名本来就是稳定的 employee_code + week_label，我已经在你当前实例上验证过：

  - GET /datasets/{dataset_id}/documents?keyword=001708 + 2026-W12 能找到目标文档
  - GET /datasets/{dataset_id}/documents/{document_id}/segments 能直接拿到周报正文

  推荐流程

  1. 代码执行 节点，统一周标签格式并生成文档名。

  from datetime import datetime, timedelta
  from urllib.parse import quote
  import re

  def main(employee_code: str, query: str) -> dict:
      today = datetime.now()

      match = re.search(r'(\d{4})-?W(\d{1,2})', query or "")
      if match:
          week_label = f"{match.group(1)}-W{int(match.group(2)):02d}"
      else:
          if '上上周' in query:
              offset = -2
          elif '上周' in query:
              offset = -1
          else:
              offset = 0
          target = today + timedelta(weeks=offset)
          year, week, _ = target.isocalendar()
          week_label = f"{year}-W{week:02d}"

      document_name = f"{employee_code} + {week_label}"
      return {
          "week_label": week_label,
          "document_name": document_name,
          "document_name_q": quote(document_name, safe="")
      }

  2. HTTP 请求 节点，查文档列表。

  - 方法：GET
  - URL：

  http://dify.amer.cn/v1/datasets/{{#env.dataset_id#}}/documents?keyword={{#<上个代码节点>.document_name_q#}}&status=available&page=1&limit=10

  - Header：

  Authorization: Bearer {{#env.dataset_key#}}

  3. 代码执行 节点，从返回结果里精确挑出目标文档。

  import json

  def main(body: str, document_name: str, employee_code: str, week_label: str) -> dict:
      payload = json.loads(body or "{}")
      for doc in payload.get("data", []):
          if doc.get("name") != document_name:
              continue

          meta = {}
          for item in doc.get("doc_metadata") or []:
              meta[item.get("name")] = str(item.get("value", ""))

          if meta.get("employee_code") == employee_code and meta.get("week_label") == week_label:
              return {
                  "found": True,
                  "document_id": doc["id"],
                  "document_name": doc["name"]
              }

      return {
          "found": False,
          "document_id": "",
          "document_name": ""
      }

  4. 条件分支

  - found = false：直接回复“未找到该员工该周周报”
  - found = true：继续

  5. HTTP 请求 节点，取这个文档的 chunks。

  - 方法：GET
  - URL：

  http://dify.amer.cn/v1/datasets/{{#env.dataset_id#}}/documents/{{#<解析文档节点>.document_id#}}/segments?status=completed&page=1&limit=100

  6. 代码执行 节点，拼接正文。

  import json

  def main(body: str) -> dict:
      payload = json.loads(body or "{}")
      rows = sorted(payload.get("data", []), key=lambda x: x.get("position", 0))
      doc_text = "\n\n".join(
          (row.get("content") or "").strip()
          for row in rows
          if (row.get("content") or "").strip()
      )
      return {"doc_text": doc_text}

  7. LLM 节点直接吃 doc_text

  以下是周报原文：
  {{#<拼接正文节点>.doc_text#}}

  基于原文回答用户问题：{{#sys.query#}}
  如果原文里没有答案，就明确说没有找到，不要编造。

  按 metadata 查怎么做
  截至我查到的 2026-03-26 官方文档，GET /datasets/{dataset_id}/documents 公开参数只有 page / limit / keyword / status，
  keyword 还是按文档名查，不是按 metadata 查。也就是说，官方公开 API 里没有“直接按 metadata 检索文档列表”的参数。
  所以如果你一定要按 metadata 查，做法是：

  1. 先分页拉文档列表
  2. 在代码节点里检查每个文档的 doc_metadata
  3. 找到 employee_code 和 week_label 都匹配的文档

  但你这里没必要走这条路，因为你已经有稳定的文档名，按 document_name 查更简单、更快。

  你这套里为什么推荐按文档名

  - 你自己生成文档时，document_name = employee_code + week_label
  - metadata 精确过滤容易被格式不一致坑到
  - segments API 取出来的是原文，不依赖语义召回，结果最稳定
