  核心问题：Dify 工作流中的时间感知

  Dify 工作流本身没有内置的"当前日期"变量，解决方案是在工作流最前端放一个 Code 节点 来计算所有时间信息。

  ---
  方案设计

  1. Code 节点：日期计算器（工作流第一个节点）

  import datetime

  today = datetime.date.today()
  weekday = today.weekday()  # 0=周一, 6=周日

  # ---- 本周区间 (周一 ~ 周日) ----
  week_start = today - datetime.timedelta(days=weekday)
  week_end = week_start + datetime.timedelta(days=6)

  # ---- 上一周区间 ----
  last_week_start = week_start - datetime.timedelta(days=7)
  last_week_end = week_start - datetime.timedelta(days=1)

  # ---- 本月区间 ----
  month_start = today.replace(day=1)
  next_month = (today.replace(day=28) + datetime.timedelta(days=4)).replace(day=1)
  month_end = next_month - datetime.timedelta(days=1)

  # ---- 上月区间 ----
  last_month_end = month_start - datetime.timedelta(days=1)
  last_month_start = last_month_end.replace(day=1)

  # ---- 周数/月份标识（用于知识库文档匹配） ----
  iso_year, iso_week, _ = today.isocalendar()
  week_label = f"{iso_year}-W{iso_week:02d}"          # 例: 2026-W13
  month_label = today.strftime("%Y-%m")                # 例: 2026-03
  last_week_label = f"{iso_year}-W{iso_week-1:02d}" if iso_week > 1 else f"{iso_year-1}-W52"
  last_month_label = last_month_start.strftime("%Y-%m")

  weekday_names = ["周一","周二","周三","周四","周五","周六","周日"]

  return {
      "today": today.isoformat(),
      "weekday_name": weekday_names[weekday],
      "weekday_num": weekday,

      # 本周
      "week_start": week_start.isoformat(),
      "week_end": week_end.isoformat(),
      "week_label": week_label,

      # 上周（生成周报时用上周的数据）
      "last_week_start": last_week_start.isoformat(),
      "last_week_end": last_week_end.isoformat(),
      "last_week_label": last_week_label,

      # 本月
      "month_start": month_start.isoformat(),
      "month_end": month_end.isoformat(),
      "month_label": month_label,

      # 上月
      "last_month_start": last_month_start.isoformat(),
      "last_month_end": last_month_end.isoformat(),
      "last_month_label": last_month_label,

      # 判断标志
      "is_friday": weekday == 4,
      "is_month_last_day": today == month_end,

      # 用于知识库查询的日期范围描述
      "week_range_desc": f"{week_start.isoformat()} 至 {week_end.isoformat()}",
      "month_range_desc": f"{month_start.isoformat()} 至 {month_end.isoformat()}"
  }

  一个业务员 = 一个文档，一周/一月 = 一个文档块（segment），每次整块替换。

  ---
  那数据怎么进去？

  你需要解决的核心问题是：每天的日报数据怎么汇入这个文档块？

  有两种思路：

  思路 A：每天"覆盖更新"整个文档块

  每天日报产生后，把本周截至目前的所有日报拼接成一整段文本，然后调用 Dify API 替换该业务员文档里的那个唯一文档块。

  周一结束 → 文档块内容 = 周一的日报
  周二结束 → 文档块内容 = 周一 + 周二的日报
  周三结束 → 文档块内容 = 周一 + 周二 + 周三的日报
  ...
  周五结束 → 文档块内容 = 周一~周五完整日报（可以生成周报了）

  对应 Dify API：

  # 更新某个文档下的指定 segment
  POST /v1/datasets/{dataset_id}/documents/{document_id}/segments/{segment_id}

  {
    "segment": {
      "content": "【2026-03-17 周一】拜访了A客户...\n【2026-03-18 周二】签约B客户...\n...",
      "keywords": ["张三", "2026-W13"]
    }
  }

  优点：简单直接，文档块始终是完整的一周数据
  缺点：需要在外部（数据库/文件）暂存每天的日报原文用于拼接

  思路 B：周末/月末一次性写入

  平时日报存在别的地方（数据库、飞书文档、Excel 等），到了周五/月末，用一个脚本或工作流：

  1. 从数据源拉取本周所有日报
  2. 拼接成一段完整文本
  3. 调用 Dify API 写入/替换文档块
  4. 然后触发周报/月报生成工作流
