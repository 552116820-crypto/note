  - 或者批量查询文档列表：
  GET /v1/datasets/{dataset_id}/documents?limit=100
  - 然后检查所有文档的 indexing_status
  - 条件判断节点：
    - 如果所有文档 indexing_status == "completed" → 跳出循环
    - 如果还有 indexing 或 parsing 状态的 → 继续循环
  - 等待：可以在代码节点中加 time.sleep(5) 等待几秒再查询，避免频繁请求（注意 Dify 代码节点默认有执行时间限制）

  循环内伪逻辑

  # 代码节点：检查索引状态
  import json

  # 解析文档列表API返回的body
  docs = json.loads(body)['data']

  # 获取迭代输出的 document_id 列表
  target_ids = set(document_ids)

  all_completed = True
  for doc in docs:
      if doc['id'] in target_ids:
          if doc['indexing_status'] != 'completed':
              all_completed = False
              break

  return {
      "all_completed": all_completed
  }

  然后条件分支判断 all_completed == True 时跳出循环。

  注意事项

  - 设置循环的最大次数（比如 30 次），防止无限循环
  - 如果 Dify 代码节点不支持 time.sleep，可以通过多做一次 HTTP 请求来间接"拖延"时间
  - 也可以考虑用 indexing_status 是否为 error 来做异常处理
