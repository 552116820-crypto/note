`items` 是 `Iteration` 节点自带的内置变量，不是你自己提前定义的。  
它表示“当前这一轮正在处理的那个数组元素”。Dify 官方文档里写的是：

- `items[object]`：当前迭代项
- `index[number]`：当前迭代序号

所以前提是你把一个数组喂给了 `Iteration`。比如 `Code` 节点返回：

```json
{
  "documents": [
    {
      "document_name": "EMP001 + 2026-W10",
      "content": "..."
    },
    {
      "document_name": "EMP001 + 2026-W11",
      "content": "..."
    }
  ]
}
```

然后你把 `documents` 这个数组接到 `Iteration` 的输入里。  
这时在 `Iteration` 里面的 `HTTP Request`、`If/Else` 等子节点里：

- `{{items.document_name}}` 就是当前文档名
- `{{items.content}}` 就是当前文档内容
- `{{index}}` 就是第几轮

如果你传进去的是字符串数组：

```json
{
  "document_names": [
    "EMP001 + 2026-W10",
    "EMP001 + 2026-W11"
  ]
}
```

那在迭代内部就应该直接用：

```jinja
{{items}}
```

不能写 `{{items.document_name}}`，因为当前项本身就是字符串，不是对象。

还有一个容易踩坑的点：`items` 只在 `Iteration` 内部可用，出了 `Iteration` 就没有这个变量了。
