  ## 方案 A（最推荐）：做一个“客户目录/索引”数据源，用工具查询

  把“有哪些客户”交给工具/接口/数据库去返回列表，LLM 只负责展示和二次筛选。

  你已经有 input.json 或源系统接口了，最简单是提供一个接口：

  - GET /customers 返回客户列表（可分页）
      - [{id, customer_name, creditNo, province, city, ...}, ...]
 
  在 Dify Workflow 里：
 
  1. 判断用户意图：如果问题像“有哪些客户/客户列表/所有客户”，走“客户目录查询”分支
  2. 用 HTTP 工具调用 /customers?page=1&page_size=200（或按你系统限制）
  3. 把结果交给模型输出（并提示“数据量大，先展示前 N 条，可按地区/名称筛选”）
  4. 用户进一步问“北京有哪些客户/名字包含xx的客户”，再用工具带过滤条件查


  ## 方案 C：把“客户名称/ID”放到文档元数据里，用元数据过滤 + 小范围检索

  有些系统支持给每个文档/块加元数据（如省份、城市、行业、客户ID），再做过滤查询。

  但注意：元数据过滤依然解决不了“列出全部文档”，它更适合：

  - “北京有哪些客户？”（先过滤 province=北京，再 TopK）
  - “行业=煤炭的客户有哪些？”（先过滤行业，再 TopK）
 
  如果你想做“有哪些客户（全部）”，还是更建议用 A 或 B。
