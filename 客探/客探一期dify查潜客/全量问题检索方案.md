  ## 方案 A（最推荐）：把“客户有哪些”做成目录查询（HTTP/代码节点）
 
  你已经明确说了：你在 Dify 里有 HTTP 请求节点拿到 body，而且你也愿意在“代码执行节点”跑 Python。

  那就直接做一个分支：

  ### 工作流路由建议
  - 如果用户问题匹配：
      - “客户有哪些 / 所有客户 / 客户列表 / 全部客户 / 有哪些公司”
  - 就不要走知识库检索（RAG），而是走：
      1. HTTP 请求节点：获取客户数据（你现在已有）
      2. 代码执行节点：从 body 解析 data，提取 customer_name（可附带信用代码、地区）
      3. 模型节点：把列表按分页展示（比如前 50 条），并提示用户如何筛选
