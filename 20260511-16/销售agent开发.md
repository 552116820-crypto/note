### Turn 4  *(2026-05-14 03:36)*

**User:**

那agent适合什么场景或者项目，给销售做智能体这可行吗

**Assistant:**

我按 `agentic-harness-patterns-zh` 的框架来回答：先判断什么任务真的需要 agent，再落到销售场景该怎么做、哪些不要做。

可行，而且**销售是比较适合做 agent 的场景之一**。但要定位清楚：一开始不要做“自动销售员”，而是做“销售副驾 / 客户开发助手 / 销售工程师助手”。

**Agent 适合什么场景**
Agent 最适合这类任务：

- 输入不固定：客户资料、官网、聊天记录、展会名录、CRM 备注、邮件、产品资料混在一起。
- 流程多步骤：查资料 → 判断客户价值 → 匹配产品 → 生成话术 → 写 CRM → 安排跟进。
- 需要用工具：搜索、CRM、邮件、日历、知识库、报价系统、企业工商信息等。
- 需要记忆：客户历史、联系人偏好、行业、已报价产品、上次沟通结论。
- 结果允许人工确认：agent 先生成建议、草稿、候选动作，人来批准。

不适合的是：规则很死的流程、一步 SQL 就能解决的报表、不能容忍错误的财务/合同/价格承诺、数据源很差又没人维护的场景。

**销售 Agent 很可行，但要分层做**
我建议从这几个能力开始：

1. **客户研究**
   输入公司名或客户名单，agent 自动查：
   - 公司业务；
   - 产品线；
   - 采购可能性；
   - 现有供应商线索；
   - 和你们产品的匹配点；
   - 推荐切入话题。

2. **线索评分**
   根据行业、规模、区域、产品匹配、融资/项目/招聘信号，给客户打分：
   - A 类：销售应马上跟；
   - B 类：进入培育；
   - C 类：暂缓。

3. **销售跟进助手**
   agent 根据客户阶段生成：
   - 微信开场白；
   - 邮件；
   - 电话提纲；
   - 拜访前 briefing；
   - 拜访后纪要；
   - 下一步跟进计划。

4. **产品匹配 / 技术销售支持**
   这类特别适合 B2B 工业品或化学品销售：
   - 客户做什么产品；
   - 可能需要你们哪类材料/助剂/油品/胶粘剂；
   - 对应卖点是什么；
   - 需要问客户哪些技术问题；
   - 哪些参数不能乱承诺。

5. **CRM 自动整理**
   会议录音、微信记录、邮件内容，让 agent 提取：
   - 客户需求；
   - 预算；
   - 竞品；
   - 决策人；
   - 下次动作；
   - 风险点；
   - 更新 CRM 字段。

**但不要一上来做这些**
不要让 agent 直接：

- 自动给客户发消息；
- 自动报价；
- 自动承诺交期、价格、性能；
- 自动修改关键 CRM 阶段；
- 自动签合同或生成正式商务条款。

这些必须有人审，至少早期必须审。

**比较靠谱的 MVP**
第一版可以做成：

> 输入客户公司名 / 客户名单 / 展会名单，输出客户画像、产品匹配、销售切入点、跟进话术和 CRM 备注草稿。

这版价值很清楚，风险也低。它主要节省销售查资料、写话术、整理纪要的时间。

第二版再接 CRM、企业微信、邮箱、日历，但所有对外动作先走“生成草稿，人工确认”。

**判断是否值得做**
如果你们销售有这些痛点，就值得做：

- 客户开发靠人工搜资料，效率低；
- 产品复杂，新销售不知道怎么切入；
- CRM 记录质量差；
- 展会后名片/照片/客户名单转化慢；
- 销售拜访前准备不足；
- 老板或销售总监想知道每个客户为什么值得跟。

所以答案是：**给销售做智能体可行，而且比你这个展会图片项目更适合 agent 化。** 但最佳形态不是“替代销售”，而是“让销售每天少做资料整理、多做判断和成交”。
