## Harness 是什么

Claude Code 的 **harness** = CLI 运行时本体：加载上下文、调度 tool、管理 turn、执行 hook、解析 settings.json、注册 slash command 和 subagent 的那套引擎代码。它是 Anthropic 发布的框架。

## Agents / Commands 的双层结构

| 层 | 例子 | 归属 |
|---|---|---|
| **机制层**（怎么发现、注册、调度 `.md` 文件） | "扫 `.claude/agents/*.md` → 解析 frontmatter → 注册成可 spawn 的 subagent"<br>"扫 `.claude/commands/*.md` → 用户敲 `/xxx` 时把内容注入主对话" | **属于 harness**（你删不掉、改不了） |
| **内容层**（`.md` 文件本身写了啥） | `exhibition-finder.md` / `run-position.md` 这些项目特定的 prompt | **不属于 harness**，是你的项目资产 |

类比 hooks：`settings.json` 里配的 `PostToolUse` 命令是你写的，执行它的 hook executor 是 harness 的。

## 实际意义

- **升级 Claude Code 版本** → harness 变，机制层可能变（比如 frontmatter schema 加新字段、加载路径调整）
- **你的 `.claude/agents/*.md` 和 `.claude/commands/*.md`** → 跟着 git 走，跨机器、跨版本都在，属于项目代码

所以 `.claude/agents/` `.claude/commands/` 这两个目录的**位置和加载规则是 harness 约定的**，但**里面的文件是你的项目内容**——就像 `package.json` 路径是 npm 约定的，但内容是你写的。

## 1. Skills 属于 harness 吗

**和 agents/commands 一样，要分双层：**

| 层 | 归属 |
|---|---|
| **机制层**：skill 在哪些路径被扫描、frontmatter schema、`/xxx` 怎么解析触发、auto-trigger 描述匹配 | **harness 的** |
| **内容层**：具体 `.md` 文件 | 看来源 |

skills 内容层有 **4 种来源**：

| 来源 | 路径 | 是不是 harness 自带 |
|---|---|---|
| Claude Code 内置 | 跟随 CLI 安装（`/init` `/review` `/security-review`） | ✅ 是，跟 harness 一起发布 |
| 你账户全局 | `~/.claude/skills/` | ❌ 你装的，可删可改 |
| 项目级 | `.claude/skills/`（你这项目没有） | ❌ 项目资产 |
| 插件 | 装插件时带进来 | ❌ 第三方 |

你刚才看到的 `grill-me` `simplify` `frontend-design` `keybindings-help` `update-config` 等——大部分是 **你账户全局装的**（`~/.claude/skills/`），不是 harness 内置；`/init` `/review` `/security-review` 才是 harness 自带。

## 2. Harness 作为用户能不能改

**源码改不了**（除非自己 fork/patch CLI 二进制），但 harness 暴露了**一大堆配置面**让你调行为：

| 你能动的 | 改啥 |
|---|---|
| `~/.claude/settings.json` + `.claude/settings.local.json` | 权限白名单、环境变量、模型、statusline |
| **hooks** | `PreToolUse` / `PostToolUse` / `Stop` / `UserPromptSubmit` 等生命周期点上跑你的 shell 命令——这是改 harness 行为最强的口子 |
| `~/.claude/keybindings.json` | 快捷键 |
| `~/.claude/skills/` `agents/` `commands/` | 加全局技能/子代理/斜杠命令 |
| `.claude/` 项目目录 | 给单项目加 skill/agent/command |
| MCP servers | 扩展 tool 表（chrome-devtools / browser-use 这些就是 MCP） |
| 插件 | 一次装一坨 |
| `CLAUDE.md` | 喂项目 / 用户级常驻 prompt |
| **memory**（你账户那个 MEMORY.md） | 跨会话记忆，自动注入 |

所以正确表述是：**harness 本体的代码改不了，但它"留出来给你改的接缝"非常多**，几乎所有日常痛点（权限弹窗、提示风格、自动化任务）都能通过这些接缝解决，不需要碰源码。

## 直观判断法

> 如果某个文件 / 配置删了 Claude Code 还能跑 → 它就**不是** harness 本体，是你的资产。
> 如果删了 Claude Code 起不来 / 行为变了 → 它是 harness 的一部分。

按这条判断：
- `.claude/agents/*.md` `.claude/commands/*.md` `~/.claude/skills/*` `settings.json` `CLAUDE.md` → **全是你的资产**
- Claude Code CLI 本体（`claude` 这个可执行文件 + 它解析这些文件的逻辑） → **harness**
