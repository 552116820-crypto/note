# figma是什么，怎么用

## Figma 简介

**Figma** 是一款基于浏览器的协作式 UI/UX 设计工具，主要用于：

- **界面设计** — 网页、移动端 App、桌面应用的 UI 设计
- **原型交互** — 创建可点击的交互原型，模拟用户操作流程
- **设计系统** — 管理组件库、设计规范（颜色、字体、间距等）
- **团队协作** — 多人实时编辑同一设计文件，类似 Google Docs

## 核心特点

| 特点 | 说明 |
|------|------|
| 浏览器端 | 无需安装，打开浏览器即可使用（也有桌面客户端） |
| 实时协作 | 多人同时编辑，实时看到对方光标和操作 |
| 组件系统 | 可复用的设计组件，修改主组件自动同步所有实例 |
| Auto Layout | 类似 CSS Flexbox 的自动布局能力 |
| Dev Mode | 开发者可直接查看标注、导出资源、复制 CSS 代码 |
| 插件生态 | 丰富的社区插件扩展功能 |

## 基本使用流程

1. **注册** — 访问 figma.com，免费账户即可开始
2. **创建文件** — 新建设计文件，选择画框（Frame）尺寸（如 iPhone、Web 等）
3. **设计** — 使用形状、文本、图片等工具进行界面设计
4. **组件化** — 将重复元素创建为组件（Component）方便复用
5. **原型** — 切换到 Prototype 标签页，添加页面间的交互连接
6. **交付** — 分享链接给开发者，开发者通过 Dev Mode 查看尺寸、颜色、间距等

## 与开发的关系

作为开发者，Figma 常见的使用场景：

- **查看设计稿** — 设计师分享链接，开发者查看标注和样式
- **导出资源** — 导出图标、图片等切图资源
- **获取 CSS** — Dev Mode 中可直接复制 CSS/Tailwind 代码
- **Figma → 代码** — 可配合插件或 AI 工具将设计稿转为前端代码

# Figma → 代码 详解

## 一、主流方案概览

| 方案 | 代表工具 | 适合场景 | 代码质量 |
|------|---------|---------|---------|
| Figma 官方 Dev Mode | Figma 内置 | 开发者手动还原 | 最高（人工写） |
| Figma 插件导出 | Locofy, Anima | 快速出页面 | 中等 |
| AI 截图转代码 | Claude, GPT-4o, v0 | 快速原型 | 中等偏上 |
| 设计系统映射 | Storybook + Figma tokens | 组件化项目 | 高 |

---

## 二、各方案详细说明

### 1. Figma Dev Mode（手动还原）

最传统也最可控的方式。开发者在 Figma 中：

- 点击元素 → 右侧面板查看 **CSS / Tailwind / Swift / Android** 代码片段
- 查看间距、颜色、字体等精确数值
- 导出图标和图片资源

```
优点：代码完全可控，质量最高
缺点：速度慢，纯手工活
适合：正式项目、对代码质量要求高的场景
```

### 2. Figma 插件自动转码

**Locofy.ai**
- 在 Figma 中标注组件类型（按钮、输入框、列表等）
- 一键导出 React / Next.js / HTML 代码
- 支持响应式布局

**Anima**
- 直接将 Figma 设计转为 React / Vue / HTML
- 支持交互动画导出
- 可同步设计更新

**Builder.io (Figma to Code 插件)**
- 支持导出 React / Vue / Svelte / Angular / HTML
- 可选 Tailwind CSS 或普通 CSS

```
优点：速度快，自动化程度高
缺点：生成的代码往往需要大量清理，嵌套层级多，命名不语义化
适合：快速出原型、Landing Page 等一次性页面
```

### 3. AI 截图/设计稿转代码（目前最热门）

这是当前发展最快的方向：

**方式 A：截图 → AI → 代码**

```
工作流程：
1. 在 Figma 中截图（或导出为 PNG）
2. 把截图给 AI（Claude / GPT-4o）
3. AI 直接生成 HTML + CSS / React + Tailwind 代码
```

例如，在 Claude Code 中你可以直接这样做：
- 把 Figma 截图保存到本地
- 让 Claude 读取截图并生成前端代码

**方式 B：v0.dev (Vercel)**
- 上传设计截图或文字描述
- 自动生成 React + Tailwind + shadcn/ui 组件
- 可在线预览和迭代调整

**方式 C：Figma → JSON → AI**
- 通过 Figma API 或插件导出设计的结构化 JSON 数据
- 把 JSON 喂给 AI，生成更精准的代码（比纯截图多了尺寸、颜色等精确信息）

```
优点：速度极快，代码可读性比插件好
缺点：像素级还原有偏差，复杂交互需手动补
适合：快速原型、中等复杂度的页面
```

### 4. 设计系统映射（企业级方案）

```
工作流程：
Figma 组件库  ←→  代码组件库（React/Vue）
     ↓                    ↓
  Design Tokens  →  CSS 变量 / Tailwind 配置
```

- **Figma Tokens 插件** → 导出颜色、字体、间距为 JSON
- **Style Dictionary** → 将 Token JSON 转为 CSS 变量 / Tailwind config
- 设计师用的 Figma 组件和开发者的代码组件一一对应

```
优点：一致性最强，长期维护成本低
缺点：前期搭建成本高
适合：中大型团队、长期迭代的产品
```

---

## 三、实际推荐的工作流

对于大多数个人开发者或小团队：

```
┌─────────────────────────────────────────┐
│  1. Figma 中完成设计                      │
│             ↓                            │
│  2. 截图/导出关键页面                      │
│             ↓                            │
│  3. 用 AI (Claude) 生成基础代码框架        │
│             ↓                            │
│  4. 回到 Figma Dev Mode 校对细节           │
│     (间距、颜色、字体精确值)               │
│             ↓                            │
│  5. 手动调整代码，达到像素级还原            │
└─────────────────────────────────────────┘
```

这个流程结合了 **AI 的速度** 和 **手动调整的精度**，是目前性价比最高的方式。

# Figma 官方不能直接导出完整可用的 HTML 页面

## Figma Dev Mode 实际给你的是这些

选中一个元素，右侧面板显示的是**片段**，比如：

```css
/* 只是这个元素的样式，不是完整页面 */
width: 320px;
height: 48px;
border-radius: 8px;
background: #4F46E5;
font-family: Inter;
font-size: 16px;
```

- 没有完整的 HTML 结构
- 没有页面布局代码
- 每个元素要一个个点开看
- 需要你自己拼装成完整页面

## 为什么不做这个功能？

因为设计稿和真实代码之间有本质差距：

| 设计稿 | 真实代码 |
|--------|---------|
| 绝对定位，固定像素 | 需要响应式、弹性布局 |
| 静态画面 | 需要交互、状态管理 |
| 一种屏幕尺寸 | 需要适配多种设备 |
| 视觉层叠 | 需要语义化 HTML 结构 |

直接按设计稿坐标导出的 HTML 基本不可用（全是 `position: absolute`，一堆嵌套 div）。

## 最实际的做法

**把 Figma 页面截图发给 AI（如 Claude）**，直接帮你生成可用的 HTML + CSS 代码。比 Figma 官方给的片段实用得多。
