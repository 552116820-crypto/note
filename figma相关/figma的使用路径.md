# Figma 有多少种使用方式，如何快速使用并进阶

## 一、四种使用入口

| 方式 | 说明 | 适合场景 |
|------|------|---------|
| **浏览器** | 直接访问 figma.com 使用 | 随时随地、轻量使用 |
| **桌面客户端** | Windows / Mac 独立应用 | 日常主力设计，性能更好 |
| **移动端 App** | Figma Mirror（手机预览） | 在真机上预览移动端设计效果 |
| **FigJam** | Figma 旗下白板工具 | 头脑风暴、流程图、需求梳理 |

对于开发者来说，**浏览器 + 桌面客户端**足够了。

---

## 二、快速上手路线（1-3天）

### Day 1：基础操作

```
必学操作（30分钟搞定）：
├── 画框 Frame（按 F）— 相当于 HTML 的 div
├── 矩形/圆形（R / O）— 基础形状
├── 文本（T）— 输入文字
├── 移动/缩放 — 鼠标拖拽 + 滚轮缩放
├── 图层面板 — 左侧，理解层级关系
└── 属性面板 — 右侧，修改颜色/尺寸/字体
```

**核心快捷键**：

| 快捷键 | 功能 |
|--------|------|
| `F` | 创建 Frame |
| `R` | 矩形 |
| `T` | 文本 |
| `V` | 选择工具 |
| `Ctrl+D` | 复制元素 |
| `Ctrl+G` | 编组 |
| `Ctrl+Shift+H` | 水平翻转 |
| `Alt+拖拽` | 复制并移动 |
| `Shift+拖拽` | 等比缩放 |

### Day 2：布局能力

```
Auto Layout（核心！）：
├── 选中元素 → Shift+A 添加 Auto Layout
├── 相当于 CSS Flexbox
├── 控制方向（横/竖）、间距、内边距
└── 这是做出"像真实网页"布局的关键
```

### Day 3：做出第一个完整页面

- 找一个你喜欢的网页，在 Figma 中临摹它
- 用 Frame + Auto Layout 搭建结构
- 不用画得完美，重点是理解**图层结构和布局逻辑**

## 三、进阶路线

### 阶段 1：组件化思维（第 1-2 周）

```
组件 Component：
├── 选中元素 → Ctrl+Alt+K 创建组件
├── 主组件修改 → 所有实例自动更新
├── Variants（变体）— 一个组件多种状态
│   例：按钮组件 = primary / secondary / disabled
└── 实例覆盖 — 实例可以单独改文字/图标，不影响主组件
```

这和前端的**组件化开发思想完全一致**，学起来很快。

### 阶段 2：设计系统（第 2-4 周）

```
Design Tokens：
├── 颜色系统 — 定义主色、中性色、语义色
├── 字体系统 — 标题/正文/标注的字号、行高、字重
├── 间距系统 — 4px 为基础单位（4/8/12/16/24/32...）
├── 圆角/阴影 — 统一规范
└── 以上全部存为 Figma 的 Local Styles 或 Variables
```

### 阶段 3：原型交互（第 3-4 周）

```
Prototype 模式：
├── 页面跳转 — 拖线连接两个 Frame
├── 过渡动画 — 滑入/淡入/Smart Animate
├── 交互触发 — 点击/悬停/拖拽
└── 可以生成可点击的演示链接分享给别人
```

### 阶段 4：与开发打通（第 4 周+）

```
Dev Mode + MCP：
├── 开启 Dev Mode 查看标注
├── 配置 Figma MCP → AI 直接读取设计数据
├── 设计稿 → Claude → 可用代码
└── 形成 设计-开发 闭环
```

## 四、学习资源推荐

| 资源 | 类型 | 说明 |
|------|------|------|
| Figma 官方教程 | 视频/文档 | 在 Figma 内点 `?` 号即可进入 |
| Figma Community | 模板 | 海量免费设计模板，直接复制学习 |
| B站搜"Figma入门" | 视频 | 中文教程，适合快速上手 |
| 临摹真实网页 | 实践 | **最有效的学习方式** |

## 五、作为开发者的最短路径

如果你的目标是**自己设计 + 自己写代码**，不需要学全部功能：

```
最小学习集：
1. Frame + Auto Layout（布局）     ← 最重要
2. 颜色 + 字体 + 间距（样式）       ← 第二重要
3. Component 基础（复用）           ← 提效
4. 导出 / 截图 → AI 生成代码        ← 产出

不需要学的：
✗ 复杂的矢量绘图（用现成图标库）
✗ 高级动效（代码实现更灵活）
✗ 团队协作管理（个人用不到）
```

**一句话总结**：学会 Frame + Auto Layout，能在 Figma 里"摆出"页面结构，剩下的交给 AI 生成代码。这是开发者用 Figma 的最高效路径。
