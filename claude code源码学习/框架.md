› Bun是它的agent框架吗


• 不是。

  Bun 在这个项目里是 JavaScript/TypeScript 运行时 + 打包/构建能力，不是 agent 框架。直接证据是：

  - README.md:13 写的是 TypeScript (Bun 运行时)
  - src/entrypoints/cli.tsx:1 和 src/main.tsx:21 都在用 bun:bundle

  这个项目的 agent 部分更像是它自己实现的一套架构，不是依赖 Bun 提供的：

  - src/coordinator/：多 Agent 协调
  - src/tasks/：任务执行模型
  - src/tools/：工具系统
  - src/assistant/、src/QueryEngine.ts：对话和推理主流程
  - src/components/ + src/ink/：终端 UI

  所以更准确地说：

  - Bun = 运行时/构建工具
  - React + Ink = CLI UI 框架
  - Agent 框架 = 这个仓库自己写的那套 coordinator/tasks/tools 架构
