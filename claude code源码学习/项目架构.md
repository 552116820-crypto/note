• 不是传统意义上的“前后端分离”项目。

  更准确地说，它是一个放在同一仓库里的“一体化 CLI/TUI 应用”：

  - “前端”是终端界面，基于 React + Ink，入口在 src/main.tsx，界面组件主要在 src/components。
  - “后端/服务层”也在同一个仓库里，包括本地任务编排、工具调用、桥接和少量服务端能力，比如 src/server、src/services、src/
    bridge。

  所以可以说“前后端都在一个仓库里”，但它不是典型的 Web 前端 + HTTP 后端架构，更像是“终端客户端 + 本地服务层 + 远程 API
  接入”的单体 TypeScript/Bun 工程。你如果愿意，我可以继续帮你把这个仓库按“UI 层 / 业务层 / 工具层 / 服务层”画成一张结构
  图。
