
#### 一、题目背景与核心要求

几何图形是小学数学与初中数学的核心知识点，需借助Claude Code工具在空白文件夹中开发一款**交互式、可视化的几何图形教学网页课件**。要求充分利用Claude Code的**Skill（技能）**、**Agent（智能体）** 调度能力，并接入**联网搜索MCP工具**，实现多模块协同开发，最终交付可直接运行的网页课件。

  

#### 二、具体开发要求

1. **功能与内容要求**
    
    1. 课件需覆盖**平面几何图形（三角形、正方形、圆形）** 和**立体几何图形（正方体、圆柱体）** 的核心知识点，包括图形定义、性质、周长/面积/体积计算公式、实例演示。
        
    2. 具备**交互式操作**：支持用户拖动调整图形参数（如三角形边长、圆的半径），实时显示计算结果；支持图形分类、拼接等互动练习。
        
    3. 界面需适配桌面端与移动端，风格简洁明了，符合中小学教学场景的视觉体验。
        
2. **Claude Code工具调用要求**
    
    1. **Skill使用**：至少使用**3个独立Skill**，且Skill需具备明确的功能边界，为课件开发提供模块化能力支撑。
        
    2. **Agent调度**：至少使用**2个不同类型的Agent**，Agent需承担独立的开发职责，并通过Claude Code实现Agent间的任务协同与结果整合。
        
    3. **工具协同**：Claude Code需作为核心调度器，合理分配任务至不同Agent，调用对应Skill完成子任务，并通过**联网搜索MCP工具**获取外部资源/数据（如最新教学大纲、图形素材、计算公式验证等）。
        
    4. **工程化规范**：开发过程需生成完整的项目结构（如HTML/CSS/JS分离、资源文件夹、配置文件），代码具备可维护性与注释，最终输出项目运行说明。
        

#### 三、关键约束与拓展要求

1. **Skill类型限定**：Skill需从**前端开发类、数据计算类、资源处理类、交互实现类、教学逻辑类**中选取，禁止重复类型。
    
2. **Agent角色限定**：Agent需具备**明确的角色定位**（如前端开发Agent、教学内容Agent、交互逻辑Agent等），不同Agent的任务需形成依赖关系（如教学内容Agent输出的知识点需作为前端开发Agent的界面渲染依据）。
    
3. **联网搜索MCP工具的应用场景**：需至少在**2个开发环节**中使用联网搜索，例如：
    
    1. 获取最新《义务教育数学课程标准》中关于几何图形的教学要求；
        
    2. 搜索几何图形的矢量素材/教学案例；
        
    3. 验证不同学段的图形计算公式表述规范。
        
4. **交付物完整性**：除网页代码外，需输出**Claude Code任务调度日志**（记录Agent调用、Skill使用、联网搜索的过程）、**项目说明文档**（含功能介绍、技术栈、运行步骤）。



#### 四、补充说明

1. 你可假设Claude Code具备以下基础能力：
    
    1. 支持自定义Skill注册与调用（如`HTML渲染Skill`、`几何计算Skill`、`素材下载Skill`）；
        
    2. 支持Agent创建与任务分配（如`前端开发Agent`负责页面编写，`教学内容Agent`负责知识点整理）；
        
    3. 联网搜索MCP工具可返回结构化的搜索结果（如JSON格式的教学大纲、图片URL列表）。
        
2. 开发过程需体现**任务拆解→Agent分配→Skill调用→结果整合**的完整流程，例如：
    
    1. 第一步：拆解任务为“教学内容整理”“前端页面开发”“交互功能实现”“素材资源获取”；
        
    2. 第二步：分配“教学内容Agent”处理知识点，“前端开发Agent”处理页面编写；
        
    3. 第三步：调用“联网搜索Skill”获取教学大纲，调用“几何计算Skill”实现公式计算；
        
    4. 第四步：整合各Agent输出结果，生成最终课件。
        

### 题目完善思路说明

1. **明确场景与目标**：将“无内容文件夹开发网页课件”细化为“中小学几何图形教学课件”，明确功能边界，避免开发方向模糊。
    
2. **强化工具调用的逻辑约束**：
    
    1. 对Skill类型、Agent角色进行分类限定，避免重复或无意义的工具使用；
        
    2. 要求Agent间形成任务依赖，体现Claude Code的调度能力，而非简单的功能堆砌。
        
3. **落地联网搜索的应用场景**：将联网搜索从“抽象要求”转化为“具体开发环节”（如教学大纲获取、素材搜索），让工具使用更具实际意义。
    
4. **补充工程化与交付要求**：增加项目结构、文档、调度日志等要求，贴近实际开发场景，考察候选人的工程化思维。
    
5. **设计评分维度**：从功能、工具调度、联网能力、工程规范、创新五个维度量化评价标准，让面试考核更具客观性。
    

此题目既考察候选人对AI工具（Claude Code）的理解与调度能力，也考察前端开发、教学内容整合、工程化等实际开发能力，同时通过联网搜索的要求考察其获取外部资源的能力，适合用于全栈开发或AI工具应用相关岗位的面试。
